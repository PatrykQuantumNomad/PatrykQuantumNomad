# Architecture

## Pattern

**Static Site Generator (SSG)** — Astro 5.3 with React islands for interactivity. No backend, no database, no auth. Fully static output deployed to GitHub Pages.

## Layers

```
┌─────────────────────────────────────────────┐
│  Pages (src/pages/)                         │  Astro routes → static HTML
├─────────────────────────────────────────────┤
│  Layouts (src/layouts/)                     │  Layout, EDALayout, GuideLayout
├─────────────────────────────────────────────┤
│  Components (src/components/)               │  .astro (server) + .tsx (React islands)
├─────────────────────────────────────────────┤
│  Libraries (src/lib/)                       │  Business logic, engines, renderers
├─────────────────────────────────────────────┤
│  Data (src/data/)                           │  JSON, TS data files, MDX content
├─────────────────────────────────────────────┤
│  Stores (src/stores/)                       │  Nanostores for client state
├─────────────────────────────────────────────┤
│  Integrations (src/integrations/)           │  Astro build hooks (IndexNow, notebooks)
├─────────────────────────────────────────────┤
│  Scripts (scripts/)                         │  Build-time: schema compilation, WASM download
└─────────────────────────────────────────────┘
```

## Interactive Tools Architecture

Four browser-based validator/analyzer tools share a common pattern:

1. **Compose Validator** — `src/lib/tools/compose-validator/`
2. **GHA Validator** — `src/lib/tools/gha-validator/`
3. **K8s Analyzer** — `src/lib/tools/k8s-analyzer/`
4. **Dockerfile Analyzer** — `src/lib/tools/dockerfile-analyzer/`

Each tool follows the same structure:
- `engine.ts` — orchestrates parsing + schema validation + rule evaluation
- `rules/` — categorized rule modules (security, best-practice, style, etc.)
- `scorer.ts` — computes scores from rule results
- `schema-validator.ts` — AJV-based schema validation
- `types.ts` — shared TypeScript types
- `url-state.ts` — LZ-string URL state encoding for sharing
- Editor panel (CodeMirror) → Engine → Results panel

## Data Flow

### Build-time
1. `prebuild` scripts: download WASM, generate graph layouts
2. Astro reads MDX content collections, data files
3. `og-image.ts` generates OG images via satori + sharp
4. `indexnow.ts` submits URLs to search engines (CI only)
5. `notebook-packager.ts` creates downloadable ZIP archives
6. Static HTML/CSS/JS output to `dist/`

### Client-side
- React islands hydrate for interactive components
- Nanostores manage cross-component state (filters, editor content, tabs)
- D3 renders SVG charts; Three.js renders 3D hero
- GSAP + Lenis handle scroll animations
- CodeMirror provides in-browser YAML editing
