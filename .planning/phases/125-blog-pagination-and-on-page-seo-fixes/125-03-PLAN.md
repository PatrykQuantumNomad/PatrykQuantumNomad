**The SIX invariants to assert:**

**Invariant 1 — Blog pagination self-canonicals (TSEO-02):**
For each of `dist/blog/2/index.html` through `dist/blog/6/index.html`, parse
the file and assert it contains literally:
```
<link rel="canonical" href="https://patrykgolabek.dev/blog/N/">
```
with `N` matching the directory number. Use a strict regex:
`/<link rel="canonical" href="https:\/\/patrykgolabek\.dev\/blog\/(\d+)\/">/`
and require `captured === N`. Fail with "blog/{N}/ missing self-canonical"
hint pointing to `src/pages/blog/[...page].astro`.

**Invariant 2 — dark-code <title> length in [55, 60] (OPSEO-01):**
Read `dist/blog/dark-code/index.html`. Extract `<title>...</title>` content
via `/<title>([^<]+)<\/title>/`. Assert `55 <= title.length <= 60`. Fail
with hint pointing to `src/data/blog/dark-code.mdx` frontmatter title.

**Invariant 3 — dark-code description ≤ 160 (OPSEO-02):**
Same file. Extract the FIRST `<meta name="description" content="...">`
value (there are multiple meta tags; match the `name="description"` one
specifically). Assert `content.length <= 160`. Fail with hint pointing to
`src/data/blog/dark-code.mdx` frontmatter description.

**Invariant 4 — Beauty Index single-language descriptions in [140, 160] (OPSEO-03):**
Enumerate `dist/beauty-index/*/index.html` EXCLUDING `dist/beauty-index/vs/`.
Use readdirSync with withFileTypes and skip any entry named `vs` or that
doesn't have an `index.html`. For each, extract the
`<meta name="description">` content and assert:
  - `content.length >= 140 AND content.length <= 160`
  - The content does NOT contain a mid-word truncation. Check the last
    visible char before any trailing `…` (or ` …`): it must be one of
    `.`, `;`, `,`, `!`, `?`, or the content must end on a word boundary
    (the preceding char was whitespace or the string has no ellipsis and
    ends on `.`/`!`/`?`). Reject patterns like `"Python's…"` or `"C#…"`
    where the char immediately before `…` is `'`, `#`, or any non-alpha
    mid-word punctuation.
  - HOWEVER: the current production rendering strategy produces a FULL
    description (length ≤ 157) WITHOUT ellipsis when the source already
    fits. In that case, `length >= 140` is the only floor — no ellipsis
    required. Distinguish the two cases by checking `content.endsWith('…')`.
Enumerate at least 26 language pages (the known corpus). Report all
failures with the slug and its length so the operator can spot the failing
language immediately.

**Invariant 5 — Dockerfile Analyzer description ≤ 160 (OPSEO-04):**
Read `dist/tools/dockerfile-analyzer/index.html`. Extract
`<meta name="description">` content. Assert `content.length <= 160`. Fail
with hint pointing to `src/pages/tools/dockerfile-analyzer/index.astro`
line 10.

**Invariant 6 — /feed.xml byte-identical to /rss.xml (TSEO-04):**
Read `dist/feed.xml` and `dist/rss.xml` as Buffers. Compute sha256 of each.
Assert `sha256(feed) === sha256(rss)`. Also assert both files exist and are
non-empty. Fail with hint pointing to `src/pages/feed.xml.ts`.

**Pattern for meta-description extraction (robust against attribute order):**

Astro serializes `<meta name="description" content="...">` with attributes
in the order declared in the template. To avoid order fragility, use TWO
passes:

```js
// Pass 1: find any <meta ...> with name="description"
const metaDescRe = /<meta\b[^>]*\bname="description"[^>]*>/g;
// Pass 2: extract content=
const contentRe = /\bcontent="([^"]*)"/;
```
Apply `metaDescRe` to the HTML, take the FIRST match, then apply `contentRe`
to that match to get the value. Do NOT use a single combined regex because
attribute order varies.

**Report schema:**

```js
{
  generatedAt: <ISO>,
  invariants: {
    paginationCanonicals: { pass: bool, details: [...] },
    darkCodeTitle: { pass: bool, length: N, title: "..." },
    darkCodeDescription: { pass: bool, length: N, description: "..." },
    beautyIndexDescriptions: { pass: bool, failed: [{slug, length, content}], total: 26 },
    dockerfileDescription: { pass: bool, length: N },
    feedRssIdentical: { pass: bool, feedSha: "...", rssSha: "..." },
  },
  overallPass: bool,
}
```

Print a single-line stdout summary when everything passes:
```
[on-page-seo] OK — 5 canonicals, dark-code title=57, 26/26 beauty-index in range, dockerfile=157, feed===rss.
```

On failure, print EACH failing invariant on its own line with the hint
before exiting with code 1.
