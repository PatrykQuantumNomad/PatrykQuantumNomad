**Content structure (7 sections):**

1. **Lede** — Opening hook: "How do you measure the beauty of a programming language?" Frame as deliberately subjective, opinionated, and fun.

2. **TldrSummary** — Quick overview: 25 languages, 6 dimensions, 1-10 per dimension, total 6-60, 4 tiers.

3. **The Six Dimensions** — Explain each dimension with its Greek symbol:
   - Phi (Φ) Aesthetic Geometry — visual cleanliness, proportional structure
   - Omega (Ω) Mathematical Elegance — inevitability, economy, algorithms from The Book
   - Lambda (Λ) Linguistic Clarity — reads like prose, signal-to-noise ratio
   - Psi (Ψ) Practitioner Happiness — flow states, community love, tooling pleasure
   - Gamma (Γ) Organic Habitability — code as a livable place, extensibility
   - Sigma (Σ) Conceptual Integrity — coherent design philosophy, features as natural consequences

   Reference the dimension descriptions from `src/lib/beauty-index/dimensions.ts`. Write in prose, do not import the data file.

4. **The Scoring Rubric** — Explain the 1-10 scale per dimension, total range 6-60. What a high score means vs a low score. Mention this is editorial judgment, not empirical measurement.

5. **The Four Tiers** — Describe each tier with score boundaries:
   - Workhorses (6-19): Languages that get the job done without aesthetic pretense
   - Practical (20-33): Functional beauty with pragmatic design choices
   - Handsome (34-46): Languages where craft and aesthetics are evident
   - Beautiful (47-60): Languages that feel like art

   Use a Callout component (type="tip") to note that tier placement does not imply language quality.

6. **Notable Rankings** — Highlight 4-5 interesting examples. Mention at least these languages by name with links to their detail pages:
   - Haskell (top Beautiful tier) — link to `/beauty-index/haskell/`
   - Rust — link to `/beauty-index/rust/`
   - Python — link to `/beauty-index/python/`
   - COBOL (bottom Workhorses tier) — link to `/beauty-index/cobol/`
   - Kotlin or Elixir — link to respective page
   Brief editorial commentary on what makes each interesting in the ranking.

7. **Explore the Index** — Closing section with links to:
   - [Beauty Index overview](/beauty-index/) — full rankings
   - [Code Comparison](/beauty-index/code/) — side-by-side code examples
   - At least 3 individual language pages already linked above

   Use a KeyTakeaway component to close with the editorial message that beauty in code is about valuing the craft of expression, not declaring a winner.

**Content tone:** First person (matching the existing blog post and site identity), confident but acknowledging subjectivity. Reference the 6 dimension data conceptually — do NOT import languages.json or any TypeScript modules.
