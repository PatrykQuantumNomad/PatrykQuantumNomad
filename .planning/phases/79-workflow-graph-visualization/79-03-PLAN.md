  <name>Task 1: Build GhaWorkflowGraph with dagre LR layout</name>
  <files>src/components/tools/gha-results/GhaWorkflowGraph.tsx</files>
  <action>
    Create the main graph component following DependencyGraph.tsx pattern. This component is default-exported for React.lazy() compatibility.

    **Module-level constants (CRITICAL -- must be outside component to avoid React Flow re-registration warnings):**
    ```typescript
    const nodeTypes = {
      'gha-trigger': GhaTriggerNode,
      'gha-job': GhaJobNode,
      'gha-step': GhaStepNode,
    };
    const edgeTypes = {
      'gha-dependency': GhaDependencyEdge,
    };
    ```

    **Layout constants:**
    - JOB_WIDTH = 240
    - JOB_HEADER_HEIGHT = 50
    - STEP_HEIGHT = 32
    - STEP_PADDING = 8
    - TRIGGER_WIDTH = 140
    - TRIGGER_HEIGHT = 50
    - STEP_PADDING_X = 8

    **Component:**
    - Read `ghaResult` from store via `useStore(ghaResult)`
    - If result is null, return empty state: "Run an analysis to see the workflow graph" (matches pre-analysis state in results tab)
    - Extract json from result by re-parsing the editor content. IMPORTANT: The ghaResult store does not carry the raw YAML text or json. The component needs access to the parsed JSON. Two approaches:
      a. Read editor content from `ghaEditorViewRef` store atom and re-parse via `parseGhaWorkflow()`
      b. Or store the json alongside the result
    - Use approach (a): read `ghaEditorViewRef.get()?.state.doc.toString()`, call `parseGhaWorkflow()`, use `json` field. If json is null (parse failed), show "Could not parse workflow" message.
    - Call `extractGhaGraphData(json, result.violations)` in a `useMemo` keyed on `[result, editorContent]`
    - Pass GhaGraphData through `layoutGraph()` function

    **layoutGraph() function (two-pass):**

    Phase 1 -- dagre layout for triggers and jobs only:
    - Create dagre Graph with `{ rankdir: 'LR', nodesep: 60, ranksep: 120 }`
    - Filter out step nodes (type !== 'step')
    - Filter out cycle edges (isCycle) before dagre to prevent infinite loops
    - Set trigger nodes: width=TRIGGER_WIDTH, height=TRIGGER_HEIGHT
    - Set job nodes: width=JOB_WIDTH, height=JOB_HEADER_HEIGHT + stepCount * STEP_HEIGHT + STEP_PADDING * 2
    - Add non-cycle needs: and trigger edges to dagre
    - Call dagre.layout(g)
    - Position trigger and job nodes from dagre output (dagre centers nodes, so subtract width/2 and height/2)

    Phase 2 -- manual positioning for step nodes:
    - For each step node, create a React Flow node with:
      - parentId: step.parentJobId
      - extent: 'parent' as const
      - position: { x: STEP_PADDING_X, y: JOB_HEADER_HEIGHT + step.stepIndex * STEP_HEIGHT }
      - type: 'gha-step'
      - data: { label, violationStatus, stepIndex }

    **Node array ordering (CRITICAL for React Flow parentId):**
    Sort final nodes array: trigger nodes first, then job nodes, then step nodes. Parent nodes MUST appear before their children in the array.

    **Job nodes as group type:**
    Job nodes need `type: 'gha-job'` and explicit `style: { width: JOB_WIDTH, height: dynamicHeight }` for the container dimensions.

    **Convert edges:**
    Map GhaGraphEdge to React Flow Edge format:
    - type: 'gha-dependency'
    - markerEnd: { type: MarkerType.ArrowClosed, color: '#888' }
    - data: { label: edge.label, isCycle: edge.isCycle }
    - Re-add cycle edges after dagre layout

    **Render:**
    - Wrap in div with className="gha-graph flex flex-col" (for CSS scoping)
    - Show cycle warning banner if hasCycle (same pattern as DependencyGraph.tsx)
    - Graph container: h-[300px] lg:h-[450px] rounded-lg overflow-hidden border
    - ReactFlow with: fitView, fitViewOptions padding 0.3, proOptions hideAttribution true, minZoom 0.3, maxZoom 2, nodesDraggable, nodesConnectable false, elementsSelectable
    - Include Controls (bottom-right, showInteractive false) and Background (dots variant)
    - Import `./gha-graph.css` for dark theme overrides

    **Do NOT import @xyflow/react/dist/style.css here** -- that import goes in GhaResultsPanel.tsx (the eager-loaded parent), not in this lazy-loaded component. This prevents style flash on lazy load.
  </action>

  </verify>
  <done>

    - Two-pass layout: dagre for triggers+jobs, manual for steps inside job containers
    - Cycle edges filtered before dagre, re-added after layout
