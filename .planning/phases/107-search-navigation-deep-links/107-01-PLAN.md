Create `src/lib/ai-landscape/graph-navigation.ts` with:

1. **`AdjacencyMap` type** — `Map<string, Set<string>>` — exported type alias.

2. **`buildAdjacencyMap(edges: Edge[]): AdjacencyMap`** — Iterates edges once (O(E)). For each edge, adds `source->target` and `target->source` entries. Both directions because the graph is navigable in either direction. Import `Edge` type from `./schema`.

3. **`nearestNeighborInDirection(currentId, direction, adjacencyMap, posMap): string | null`** —
   - `direction` is `'up' | 'down' | 'left' | 'right'`
   - For each neighbor of `currentId` in the adjacency map, compute the angle from current position to neighbor position using `Math.atan2(dy, dx)`
   - Direction vectors: up = -PI/2, down = PI/2, left = PI, right = 0
   - Score each neighbor using cosine similarity: `Math.cos(angle - targetAngle)`
   - Return the neighbor with the highest score where score > 0 (within 90 degrees of desired direction)
   - **Dead-end fallback (critical — prevents focus getting stuck):** If no neighbor scores > 0, return the neighbor with the best score regardless (i.e., the nearest neighbor in any direction). Only return null if the node has zero neighbors.

Parameters:
```typescript
currentId: string
direction: 'up' | 'down' | 'left' | 'right'
adjacencyMap: AdjacencyMap
posMap: Map<string, { x: number; y: number }>
```

Use 2-space indent, single quotes, no default export. Match existing convention in ancestry.ts and graph-data.ts (named exports, JSDoc on exported functions).

Create a search autocomplete component with ARIA combobox pattern.

Props interface:
```typescript
interface SearchBarProps {
  nodes: AiNode[];
  onSelect: (node: AiNode) => void;
}
```

State: `query` (string), `isOpen` (boolean), `activeIndex` (number, -1 = no active item).

Filtering: `useMemo` — if query is empty, return []. Otherwise, `nodes.filter(n => n.name.toLowerCase().includes(query.toLowerCase())).slice(0, 8)`.

Keyboard handling within the search input (NOT the graph keyboard handler):
- ArrowDown: increment activeIndex (cap at filtered.length - 1), preventDefault
- ArrowUp: decrement activeIndex (min 0), preventDefault
- Enter + activeIndex >= 0: call `onSelect(filtered[activeIndex])`, clear query, close dropdown, preventDefault
- Escape: clear query, close dropdown, blur the input (`inputRef.current?.blur()`)

Rendering:
- Container: `<div className="relative">`
- Input: `type="search"`, `role="combobox"`, `aria-expanded={isOpen && filtered.length > 0}`, `aria-controls="search-listbox"`, `aria-activedescendant` pointing to active option, `aria-autocomplete="list"`, `aria-label="Search AI landscape concepts"`, `placeholder="Search concepts..."`. Use CSS custom properties for theming: `bg-[var(--color-surface)]`, `border-[var(--color-border)]`, `text-[var(--color-text-primary)]`. Add `rounded-lg border px-3 py-2 text-sm w-full` Tailwind classes. Add a `ref={inputRef}` using `useRef<HTMLInputElement>`.
- onChange: set query, open dropdown, reset activeIndex to -1
- onFocus: reopen if query has text
- onBlur: `setTimeout(() => setIsOpen(false), 150)` (delay allows click on dropdown item)
- Dropdown: `<ul id="search-listbox" role="listbox">` — only rendered when `isOpen && filtered.length > 0`. Positioned `absolute z-20 mt-1 w-full`. Styled with surface bg, border, rounded-lg, shadow-lg, max-h-60, overflow-y-auto.
- Each item: `<li role="option" aria-selected={i === activeIndex}>` — `onMouseDown` (not onClick, to fire before blur) calls `onSelect(node)`, clears query, closes. Highlight active item with `bg-[var(--color-accent)]/10`. Show `node.name` as text, with `node.cluster` as a small secondary label.
