  <name>GHA Category-Weighted Scoring Engine</name>
  <files>src/lib/tools/gha-validator/scorer.ts, src/lib/tools/gha-validator/types.ts, src/lib/tools/gha-validator/__tests__/scorer.test.ts</files>
  <behavior>
    SCORE-01 Category weights:
    - Schema: 15%, Security: 35%, Semantic: 20%, Best Practice: 20%, Style: 10%
    - 'actionlint' category violations excluded from scoring (0% weight, skip entirely)

    SCORE-02 Overall score with letter grade:
    - No violations -> score 100, grade "A+"
    - 1 security error -> deducts 15 points from security category (100-15=85), weighted at 35% -> overall = 0.15*100 + 0.35*85 + 0.20*100 + 0.20*100 + 0.10*100 -> 94.75 -> grade "A"
    - Diminishing returns: 2nd violation in same category deducts less: 15 / (1 + 0.3*1) = 11.54
    - Grade thresholds: 97+ A+, 93+ A, 90+ A-, 87+ B+, 83+ B, 80+ B-, 77+ C+, 73+ C, 70+ C-, 67+ D+, 63+ D, 60+ D-, <60 F

    SCORE-03 Per-category sub-scores:
    - Each category starts at 100, deductions reduce it (floor at 0)
    - CategoryScore includes: category, score, weight, deductions[]
    - Aggregate = sum(categoryScore * weight / 100) rounded to integer

    Edge cases:
    - Empty violations array -> score 100, grade "A+", all categories at 100
    - 'actionlint' category violations (GA-L017, GA-L018, GA-L000) -> skipped, not counted
    - Many violations in one category -> category score floors at 0, doesn't go negative
  </behavior>
  <implementation>
    1. Add to types.ts: GhaScoreDeduction, GhaCategoryScore, GhaScoreResult interfaces
    2. Create scorer.ts with CATEGORY_WEIGHTS (Schema 15, Security 35, Semantic 20, Best Practice 20, Style 10), SEVERITY_DEDUCTIONS (error 15, warning 8, info 3), SCORED_CATEGORIES array (excludes 'actionlint'), computeGhaScore function, computeGrade function
    3. Port directly from src/lib/tools/dockerfile-analyzer/scorer.ts pattern but with GHA-specific category weights. Key difference: GHA scorer reads severity directly from GhaUnifiedViolation (no rule lookup needed -- violations already carry severity and category). This is simpler than the Dockerfile scorer which needs a ruleLookup map.
    4. computeGhaScore accepts GhaUnifiedViolation[] and returns GhaScoreResult
  </implementation>
