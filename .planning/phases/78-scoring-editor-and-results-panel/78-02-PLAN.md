<task type="auto">
  <name>Task 2: Build GhaEditorPanel with two-pass orchestration</name>
  <files>src/components/tools/GhaEditorPanel.tsx</files>
  <action>
    Port from `src/components/tools/K8sEditorPanel.tsx` with two-pass GHA-specific orchestration:

    **Imports:** useCodeMirrorYaml from hook, runPass1/mergePass2 from engine, allGhaRules/getGhaRuleById from rules/index, computeGhaScore from scorer, createActionlintWorker from worker-client, SAMPLE_GHA_WORKFLOW from sample-workflow, ghaResult/ghaAnalyzing/ghaResultsStale/ghaWasmProgress/ghaWasmLoading from store, setDiagnostics from @codemirror/lint, FullscreenToggle from results.

    **Two-pass analyze function:**
    1. Get YAML from editor view state
    2. If empty, clear diagnostics and result, return
    3. Set ghaResultsStale(false), ghaAnalyzing(true)
    4. Track a `generationRef` counter -- increment on each analyze call
    5. **Pass 1 (synchronous):**
       - `const pass1 = runPass1(yaml, allGhaRules)`
       - `const pass1Score = computeGhaScore(pass1.violations)`
       - `ghaResult.set({ violations: pass1.violations, score: pass1Score, pass: 1 })`
       - Convert violations to CodeMirror Diagnostics (same toDiagnostics pattern as K8sEditorPanel)
       - `view.dispatch(setDiagnostics(view.state, diagnostics))`
       - Set ghaAnalyzing(false) -- Pass 1 is complete, UI responsive
    6. **Pass 2 (async WASM Worker):**
       - If workerRef.current is null (first analyze), create Worker via createActionlintWorker:
         - onReady: set wasmReadyRef=true, call worker.analyze(yaml)
         - onResult: check generation matches; if so, merge results via mergePass2(pass1Violations, errors), recompute score, update ghaResult atom, update CM diagnostics
         - onError: set ghaWasmError atom
         - onProgress: set ghaWasmProgress atom
       - Else if wasmReadyRef is true (subsequent): just call worker.analyze(yaml)
       - Store current generation + pass1 violations in refs so onResult callback can check staleness

    **toDiagnostics helper (inline in file):**
    Convert GhaUnifiedViolation[] to Diagnostic[] -- clamp line numbers to doc.lines, use severity directly from violation (no rule lookup needed -- GhaUnifiedViolation already carries severity). Message format: `[${v.ruleId}] ${v.message}`.

    **Layout:**
    - Same card layout as K8sEditorPanel (rounded border, header with title + buttons)
    - Title: "Editor" with "Paste or edit your GitHub Actions workflow YAML" subtitle
    - Analyze button (blue/accent, triggers analyze) + WASM progress indicator below button when loading
    - Keyboard shortcut hint: "Cmd+Enter" / "Ctrl+Enter" based on navigator.platform
    - FullscreenToggle component in header
    - Editor container div with containerRef from hook

    **Props:** `{ onToggleFullscreen?: () => void; isFullscreen?: boolean }` (same as K8sEditorPanel)

    **Cleanup:** On component unmount, terminate Worker if exists (workerRef.current?.terminate()).

    **CRITICAL anti-patterns to avoid:**
    - Do NOT use `linter()` extension -- use `setDiagnostics()` only (button-triggered, not auto-triggered)
    - Do NOT await WASM before showing Pass 1 results
    - Do NOT create a new Worker on every analyze click -- reuse workerRef
    - Do NOT push stale Pass 2 diagnostics -- check generation counter
  </action>
