Create `src/components/beauty-index/VsFaqJsonLd.astro` mirroring the pattern of the existing `VsJsonLd.astro` (read that file first to match style).

Props:
```typescript
interface Props {
  faq: { question: string; answer: string }[];
}
```

Build the schema object:
```typescript
const schema = {
  '@context': 'https://schema.org',
  '@type': 'FAQPage',
  mainEntity: faq.map((f) => ({
    '@type': 'Question',
    name: f.question,
    acceptedAnswer: {
      '@type': 'Answer',
      text: f.answer,
    },
  })),
};
```

Render as:
```astro
<script type="application/ld+json" set:html={JSON.stringify(schema)} />
```

Answer text MUST be plain text (no HTML). The lib's `stripHtml` already guarantees this — do NOT add an additional sanitization pass here to avoid double-escaping. Trust the contract with `buildVsContent`.

Do NOT wrap in extra HTML, do NOT add `is:inline`. The output is a single `<script>` tag injected into the page head (or body — Google accepts both).

Note from research: Google's April 2026 FAQ rich-result rules restrict SERP rich snippets to gov/health sites. This component still ships because the schema provides AI-search citation value (Perplexity, ChatGPT Search) and featured-snippet eligibility. Rationale documented in RESEARCH.md Pattern 4.
