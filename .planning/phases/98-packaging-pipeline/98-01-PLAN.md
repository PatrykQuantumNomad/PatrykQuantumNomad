<task type="auto" tdd="true">
  <name>Task 1: TDD core ZIP packaging functions with archiver</name>
  <files>src/lib/eda/notebooks/packager.ts, src/lib/eda/notebooks/__tests__/notebook-packager.test.ts</files>
  <behavior>
    **Install archiver first:** `npm install archiver @types/archiver`

    **ZipEntry type:**
    - Has `name` (string) and either `content` (string) or `sourcePath` (string)

    **createZipFile(outputPath, entries):**
    - Creates a ZIP archive at `outputPath` using archiver (locked decision, NOT JSZip)
    - Uses `archiver('zip', { zlib: { level: 9 } })` for maximum compression
    - For entries with `content`: uses `archive.append(content, { name })`
    - For entries with `sourcePath`: uses `archive.file(sourcePath, { name })`
    - Returns a Promise that resolves on the output stream's `close` event (NOT on finalize() -- see research pitfall #1)
    - Rejects on archive `error` event
    - Rejects on archive `warning` event (unless `err.code === 'ENOENT'`)
    - Creates parent directories with `mkdirSync(dirname(outputPath), { recursive: true })`

    **buildNotebookZipEntries(slug, projectRoot):**
    - Takes a case study slug and project root path
    - Looks up config via `getCaseStudyConfig(slug)` -- throws if not found
    - Builds notebook via `buildStandardNotebook(slug)`
    - Serializes notebook as `JSON.stringify(notebook, null, 1) + '\n'` (1-space indent, trailing newline per research)
    - Reads .DAT file from `{projectRoot}/handbook/datasets/{config.dataFile}` as UTF-8
    - Normalizes .DAT content: `.replace(/\r\n/g, '\n')` to ensure LF-only line endings
    - Returns array of 3 ZipEntry objects:
      1. `{ name: '{slug}.ipynb', content: notebookJson }`
      2. `{ name: '{config.dataFile}', content: normalizedDatContent }`
      3. `{ name: 'requirements.txt', content: REQUIREMENTS_TXT }`

    **Test cases:**
    - createZipFile produces a valid ZIP file at the specified path
    - createZipFile ZIP contains all provided entries (verify with extracting/reading the ZIP)
    - createZipFile creates parent directories if they don't exist
    - buildNotebookZipEntries returns exactly 3 entries for a valid slug
    - buildNotebookZipEntries entry[0] name is `{slug}.ipynb`
    - buildNotebookZipEntries entry[0] content is valid JSON with nbformat 4 and nbformat_minor 5
    - buildNotebookZipEntries entry[0] content uses 1-space indentation (check first line indent)
    - buildNotebookZipEntries entry[0] content ends with newline
    - buildNotebookZipEntries entry[1] name matches the config dataFile (e.g., 'RANDN.DAT')
    - buildNotebookZipEntries entry[1] content has no CRLF (no \r\n anywhere)
    - buildNotebookZipEntries entry[2] name is 'requirements.txt'
    - buildNotebookZipEntries entry[2] content equals REQUIREMENTS_TXT constant
    - buildNotebookZipEntries throws for unknown slug
  </behavior>
