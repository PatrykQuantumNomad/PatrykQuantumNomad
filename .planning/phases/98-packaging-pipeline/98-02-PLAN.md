  <action>
    Create `src/integrations/notebook-packager.ts` following the existing `indexnow.ts` pattern:

    1. Import `AstroIntegration` type from 'astro', `fileURLToPath` from 'node:url', `join` from 'node:path', `mkdirSync` from 'node:fs'
    2. Import `createZipFile`, `buildNotebookZipEntries` from '../lib/eda/notebooks/packager'
    3. Import `STANDARD_SLUGS` from '../lib/eda/notebooks/templates/standard'
    4. Export default function `notebookPackager()` returning `AstroIntegration`
    5. Integration name: `'notebook-packager'`
    6. Single hook: `'astro:build:done': async ({ dir, logger })`
    7. In the hook:
       - Convert `dir` to filesystem path: `const distPath = fileURLToPath(dir)`
       - Build output directory: `const outDir = join(distPath, 'downloads', 'notebooks')`
       - Create directory: `mkdirSync(outDir, { recursive: true })`
       - Record start time: `const start = Date.now()`
       - Loop over `STANDARD_SLUGS`, for each slug:
         - Call `buildNotebookZipEntries(slug, process.cwd())` to get entries
         - Call `await createZipFile(join(outDir, \`${slug}.zip\`), entries)`
       - Log elapsed time: `logger.info(\`Packaged ${STANDARD_SLUGS.length} notebooks in ${((Date.now() - start) / 1000).toFixed(2)}s\`)`

    Then update `astro.config.mjs`:
    1. Add import: `import notebookPackager from './src/integrations/notebook-packager';`
    2. Add `notebookPackager()` to the integrations array between `indexNow()` and `react()` on line 112
       - Change: `}), indexNow(), react()],`
       - To: `}), indexNow(), notebookPackager(), react()],`

    Design note: Using STANDARD_SLUGS directly (not a builder registry map) because Phase 100 will extend this when it adds advanced builders. The current integration is intentionally simple for the 7 standard slugs. Phase 100 can add its 3 advanced slugs by importing them and extending the loop. Keeping it simple now avoids over-engineering.
