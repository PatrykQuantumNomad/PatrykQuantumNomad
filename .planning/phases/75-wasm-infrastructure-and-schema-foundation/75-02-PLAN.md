**2. Create `src/lib/tools/gha-validator/parser.ts`:**

YAML parsing module for GitHub Actions workflows. Adapt from `src/lib/tools/compose-validator/parser.ts` but simpler (no merge keys, no interpolation normalization):

```typescript
import { parseDocument, LineCounter, isMap, isPair, isScalar, isSeq } from 'yaml';
import type { Document } from 'yaml';

// Import GhaRuleViolation from types.ts (created in Plan 01)
import type { GhaRuleViolation } from './types';

export interface GhaParseResult {
  doc: Document;
  lineCounter: LineCounter;
  json: Record<string, unknown> | null;
  parseSuccess: boolean;
  parseErrors: GhaRuleViolation[];
}

export function parseGhaWorkflow(rawText: string): GhaParseResult {
  const lineCounter = new LineCounter();

  const doc = parseDocument(rawText, {
    lineCounter,
    prettyErrors: true,
    uniqueKeys: false,
    keepSourceTokens: false,
  });

  // Map YAML syntax errors to GA-S001 violations
  const parseErrors: GhaRuleViolation[] = doc.errors.map((err) => {
    let line = 1;
    let col = 1;
    if (err.pos && Array.isArray(err.pos) && err.pos.length >= 1) {
      try {
        const pos = lineCounter.linePos(err.pos[0]);
        line = pos.line;
        col = pos.col;
      } catch { /* keep defaults */ }
    }
    return { ruleId: 'GA-S001', line, column: col, message: err.message };
  });

  const parseSuccess = doc.errors.length === 0;

  let json: Record<string, unknown> | null = null;
  if (parseSuccess) {
    try {
      const rawJson = doc.toJSON();
      if (rawJson && typeof rawJson === 'object' && !Array.isArray(rawJson)) {
        json = rawJson as Record<string, unknown>;
      }
    } catch { /* leave as null */ }
  }

  return { doc, lineCounter, json, parseSuccess, parseErrors };
}
```

Note: Do NOT use `version: '1.1'` or `merge: true` -- GitHub Actions workflows are standard YAML 1.2, unlike Docker Compose which requires YAML 1.1 for merge keys.

Also export `resolveInstancePath` and `getNodeLine` helper functions. These should be copied from `src/lib/tools/compose-validator/parser.ts` (they are generic YAML AST utilities). The functions are identical -- they walk the YAML AST following a JSON Pointer path to resolve line numbers from ajv error instancePaths.

**1. Create `src/lib/tools/gha-validator/schema-validator.ts`:**

Follow the pattern from `src/lib/tools/compose-validator/schema-validator.ts`. Two public functions:

`validateGhaSchema(json)` -- raw ajv validation returning ErrorObject[]
`categorizeSchemaErrors(errors, doc, lineCounter)` -- maps ajv errors to GA-S001 through GA-S008

**Schema rule mappings (8 rules):**

| Rule ID | Name | ajv keyword | Description |
|---------|------|-------------|-------------|
| GA-S001 | YAML Syntax Error | (from parser) | Malformed YAML -- covered by parseErrors in parser.ts |
| GA-S002 | Unknown Property | `additionalProperties` | Unrecognized key in workflow/job/step |
| GA-S003 | Type Mismatch | `type` | Wrong value type (e.g., string where boolean expected) |
| GA-S004 | Missing Required Field | `required` | Required property missing (e.g., `runs-on` in job) |
| GA-S005 | Invalid Enum Value | `enum` | Value not in allowed set (e.g., invalid event type) |
| GA-S006 | Invalid Format | `oneOf` / `anyOf` | Value doesn't match any allowed format variant |
| GA-S007 | Pattern Mismatch | `pattern` | Value doesn't match expected regex pattern |
| GA-S008 | Invalid Structure | (fallback) | Generic schema error not matching a specific rule |

Implementation for `categorizeSchemaErrors`:

```typescript
import type { ErrorObject, ValidateFunction } from 'ajv';
import type { Document, LineCounter } from 'yaml';
import { validate as _validate } from './validate-gha.js';
import { resolveInstancePath, getNodeLine } from './parser';
import type { GhaRuleViolation } from './types';

const validate = _validate as ValidateFunction;

export function validateGhaSchema(json: Record<string, unknown>): ErrorObject[] {
  const valid = validate(json);
  if (valid || !validate.errors) return [];
  return validate.errors;
}

export function categorizeSchemaErrors(
  errors: ErrorObject[],
  doc: Document,
  lineCounter: LineCounter,
): GhaRuleViolation[] {
  const raw: { violation: GhaRuleViolation; instancePath: string }[] = [];
  const specificPaths = new Set<string>();

  for (const error of errors) {
    const categorised = categoriseSingleError(error, doc, lineCounter);
    if (categorised) {
      raw.push({ violation: categorised, instancePath: error.instancePath });
      if (categorised.ruleId !== 'GA-S006') {
        specificPaths.add(error.instancePath);
      }
    }
  }

  // Suppress fallback oneOf/anyOf errors when a specific rule matched at the same path
  const filtered = raw
    .filter(({ violation, instancePath }) => {
      if (violation.ruleId !== 'GA-S006') return true;
      return !specificPaths.has(instancePath);
    })
    .map(({ violation }) => violation);

  // Deduplicate: same ruleId + line + message
  const seen = new Set<string>();
  const deduplicated: GhaRuleViolation[] = [];
  for (const v of filtered) {
    const key = `${v.ruleId}:${v.line}:${v.message}`;
    if (!seen.has(key)) {
      seen.add(key);
      deduplicated.push(v);
    }
  }

  return deduplicated;
}
```

The `categoriseSingleError` function maps ajv error keywords:
- `additionalProperties` -> GA-S002 with "Unknown property '{prop}' in {context}" message
- `type` -> GA-S003 with "Expected {type} but got {actual}" message
- `required` -> GA-S004 with "Missing required property '{prop}'" message
- `enum` -> GA-S005 with "Invalid value. Must be one of: {values}" message
- `oneOf` or `anyOf` -> GA-S006 with "Value does not match any expected format" message
- `pattern` -> GA-S007 with "Value does not match expected pattern" message
- Default fallback -> GA-S008 with humanized ajv message

Include a `humanizeAjvError` function matching the compose validator pattern.

For instancePath context: extract meaningful context from the path (e.g., `/jobs/build/steps/0` -> "in job 'build', step 1") to make messages more helpful than raw JSON Pointer paths.
