**1. Create `src/lib/tools/gha-validator/worker/actionlint-worker.ts`:**

Classic Web Worker that loads the Go WASM binary with progress tracking.

CRITICAL implementation details from research:
- Use `importScripts('/wasm/wasm_exec.js')` at the top (Classic Worker pattern, locked decision)
- Set global callbacks on `self` BEFORE calling `go.run(instance)`:
  - `self.onCheckCompleted = (errors) => postMessage({ type: 'result', payload: errors })`
  - `self.showError = (msg) => postMessage({ type: 'error', payload: msg })`
  - `self.dismissLoading = () => postMessage({ type: 'ready' })` -- this signals WASM is initialized
  - `self.getYamlSource = () => ''` -- required by Go binary but not used (we pass YAML via runActionlint)
- DO NOT `await go.run(instance)` -- it returns a Promise that NEVER resolves (Go uses `select {}`)
  Just call `go.run(instance)` and let it run. Chain readiness off the `dismissLoading` callback.
- For progress tracking: use `fetch('/wasm/actionlint.wasm')` then `response.body.getReader()` to stream
  chunks while reporting `{ received, total }` via postMessage. Concatenate chunks into a single
  Uint8Array buffer, then call `WebAssembly.instantiate(buffer, go.importObject)`.
- `self.onmessage` handler: when `type === 'analyze'`, call `(self as any).runActionlint(payload)`.
  The Go binary sets `runActionlint` on the global scope via `js.Global().Set()`.
- Wrap `init()` in a try/catch and post `{ type: 'error' }` on failure.
- TypeScript: use `declare const Go: any;` at the top since `wasm_exec.js` defines it globally.
  Also `declare function importScripts(...urls: string[]): void;` for the Classic Worker API.

```typescript
// Approximate structure (adapt for exact typing):
declare const Go: any;
declare function importScripts(...urls: string[]): void;

importScripts('/wasm/wasm_exec.js');

const go = new Go();

// Set callbacks that Go WASM binary expects on globalThis
(self as any).onCheckCompleted = (errors: any[]) => {
  self.postMessage({ type: 'result', payload: errors });
};
(self as any).showError = (msg: string) => {
  self.postMessage({ type: 'error', payload: msg });
};
(self as any).dismissLoading = () => {
  self.postMessage({ type: 'ready' });
};
(self as any).getYamlSource = () => '';

async function init() {
  const response = await fetch('/wasm/actionlint.wasm');
  const reader = response.body!.getReader();
  const contentLength = +(response.headers.get('Content-Length') ?? 0);

  const chunks: Uint8Array[] = [];
  let received = 0;
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    chunks.push(value);
    received += value.length;
    self.postMessage({ type: 'progress', payload: { received, total: contentLength } });
  }

  const buffer = new Uint8Array(received);
  let offset = 0;
  for (const chunk of chunks) {
    buffer.set(chunk, offset);
    offset += chunk.length;
  }

  const result = await WebAssembly.instantiate(buffer, go.importObject);
  // DO NOT await -- go.run() never resolves (Go uses select{} to stay alive)
  go.run(result.instance);
  // dismissLoading callback fires -> posts { type: 'ready' }
}

self.onmessage = ({ data }) => {
  if (data.type === 'analyze') {
    (self as any).runActionlint(data.payload);
  }
};

init().catch(err => {
  self.postMessage({ type: 'error', payload: String(err) });
});
```

**2. Create `src/lib/tools/gha-validator/worker/worker-client.ts`:**

Typed wrapper for the Worker, used by the main thread. Follow the pattern established in the compose/K8s validators where store updates are done by the consumer, not the client itself.

```typescript
import type { ActionlintError, WasmProgress, WorkerOutMessage } from '../types';

export interface ActionlintWorkerCallbacks {
  onReady: () => void;
  onResult: (errors: ActionlintError[]) => void;
  onError: (msg: string) => void;
  onProgress: (progress: WasmProgress) => void;
}

export function createActionlintWorker(callbacks: ActionlintWorkerCallbacks) {
  const worker = new Worker(
    new URL('./actionlint-worker.ts', import.meta.url),
    // Omit { type: 'module' } -- Classic Worker per locked decision
  );

  worker.onmessage = ({ data }: MessageEvent<WorkerOutMessage>) => {
    switch (data.type) {
      case 'ready': callbacks.onReady(); break;
      case 'result': callbacks.onResult(data.payload); break;
      case 'error': callbacks.onError(data.payload); break;
      case 'progress': callbacks.onProgress(data.payload); break;
    }
  };

  worker.onerror = (event) => {
    callbacks.onError(event.message || 'Worker initialization failed');
  };

  return {
    analyze: (yaml: string) => worker.postMessage({ type: 'analyze', payload: yaml }),
    terminate: () => worker.terminate(),
  };
}
```

The client returns `{ analyze, terminate }`. The consumer (future React component or console test) calls `createActionlintWorker()` with callbacks that update nanostores.
