    **2. Modify InteractiveGraph.tsx to add compare mode:**

    Add new state:
    ```typescript
    const [compareMode, setCompareMode] = useState(false);
    const [compareNode, setCompareNode] = useState<AiNode | null>(null);  // second node for comparison
    ```

    Add compare toggle button:
    - Render a "Compare" toggle button in the toolbar area (near the zoom reset button or in the SearchBar row)
    - When clicked: `setCompareMode(prev => !prev)`. When toggled off: `setCompareNode(null)`
    - Visual state: button highlighted/active when compareMode is true
    - Button text: "Compare" when off, "Cancel Compare" when on (or use an icon toggle)
    - Disable compare button during active tour: `disabled={tour.isActive}`

    Modify handleNodeClick for compare mode:
    - When `compareMode` is true and `selectedNode` is already set:
      - If the clicked node is the same as selectedNode: ignore (can't compare a node with itself)
      - Otherwise: `setCompareNode(clickedNode)` and highlight both nodes:
        ```typescript
        setHighlightedNodeIds(new Set([selectedNode.id, clickedNode.id]));
        ```
    - When `compareMode` is true and `selectedNode` is null:
      - Set `selectedNode` normally (first node of comparison)
    - When `compareMode` is false: existing behavior (set selectedNode, clear highlights)

    Replace DetailPanel rendering with compare-aware logic:
    ```tsx
    // Desktop side panel
    {isDesktop && compareMode && selectedNode && compareNode && (
      <div className="w-96 shrink-0 border-l border-[var(--color-border)] bg-[var(--color-bg-primary)]">
        <ComparePanel
          node1={selectedNode}
          node2={compareNode}
          edges={edges}
          nodeMap={nodeMap}
          onClose={() => { setCompareMode(false); setCompareNode(null); }}
        />
      </div>
    )}
    {isDesktop && !(compareMode && compareNode) && selectedNode && (
      <div className="w-80 shrink-0 border-l border-[var(--color-border)] bg-[var(--color-bg-primary)]">
        <DetailPanel ... />
      </div>
    )}
    ```
    - Note: ComparePanel gets `w-96` (wider than DetailPanel's `w-80`) because it shows two columns
    - Mobile: ComparePanel renders inside BottomSheet when `!isDesktop`

    Add compare mode hint text:
    - When compareMode is true and selectedNode is set but compareNode is null:
      - Show a hint bar above the graph: "Click another node to compare with {selectedNode.name}"
    - When compareMode is true and both nodes are selected: hide the hint

    Handle compare mode exit:
    - Clicking the X on ComparePanel: `setCompareMode(false); setCompareNode(null); setHighlightedNodeIds(new Set());`
    - Pressing Escape during compare mode: exit compare (add to keyboard handler)
    - Starting a tour: exit compare mode (add to tour.start flow)

    **Important:**
    - Compare mode and tour mode are MUTUALLY EXCLUSIVE. Starting a tour exits compare mode. Activating compare mode is blocked during tours.
    - ComparePanel REPLACES DetailPanel when both comparison nodes are selected (per research pitfall 4). They never render simultaneously.
    - Compare mode is ephemeral -- no URL state for compare (per research recommendation). Only VS pages have permanent URLs.
    - Do NOT add URL parameters for compare mode.
