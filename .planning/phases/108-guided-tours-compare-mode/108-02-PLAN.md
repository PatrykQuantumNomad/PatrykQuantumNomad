  <action>
    Create `src/components/ai-landscape/useTour.ts`:
    - Import `Tour`, `TOURS` from `../../lib/ai-landscape/tours`
    - Export a `useTour` hook that returns:
      ```typescript
      {
        activeTour: Tour | null;        // null when idle
        currentStep: number;            // 0-indexed step index
        isActive: boolean;              // shorthand for activeTour !== null
        start: (tourId: string) => void;
        next: () => void;               // clamps to last step (does not auto-exit)
        prev: () => void;               // clamps to 0
        exit: () => void;               // returns to idle
        currentNodeId: string | null;   // activeTour?.steps[currentStep].nodeId
        narrative: string | null;        // activeTour?.steps[currentStep].narrative
        highlightNodeIds: Set<string>;  // current + prev + next tour step nodeIds for edge visibility
      }
      ```
    - `start(tourId)`: finds tour by id in TOURS, sets activeTour and currentStep=0
    - `next()`: increments currentStep, clamped to steps.length - 1
    - `prev()`: decrements currentStep, clamped to 0
    - `exit()`: sets activeTour to null, currentStep to 0
    - `highlightNodeIds`: computed set containing current step nodeId PLUS the previous step nodeId (if exists) PLUS the next step nodeId (if exists). This causes edges connecting adjacent tour steps to be highlighted via the existing `bothInChain` opacity logic in InteractiveGraph.
    - Use useState for activeTour and currentStep; useMemo for derived values (currentNodeId, narrative, highlightNodeIds)

    Create `src/components/ai-landscape/TourSelector.tsx`:
    - Import `TOURS` from `../../lib/ai-landscape/tours`
    - Props: `{ onStartTour: (tourId: string) => void }`
    - Render a row of 3 tour cards (or a compact dropdown on mobile -- use a simple button group or card layout)
    - Each card shows: tour title, tour description, step count, and a "Start Tour" button
    - Style with Tailwind: `flex gap-2 flex-wrap` container, each card `rounded-lg border p-3 cursor-pointer hover:border-accent` (use `var(--color-accent)` or Tailwind equivalent)
    - Use project font family: `font-family: 'DM Sans', sans-serif` via Tailwind's font-sans or inline style
    - Keep it simple -- this is a selection UI, not a marketing page

    Create `src/components/ai-landscape/TourBar.tsx`:
    - Props: `{ tour: Tour; currentStep: number; narrative: string; onNext: () => void; onPrev: () => void; onExit: () => void; totalSteps: number }`
    - Layout: horizontal bar with [Exit X] | Tour Title | "Step {n+1} of {total}" | narrative text | [< Prev] [Next >]
    - Prev button disabled when currentStep === 0
    - Next button disabled when currentStep === totalSteps - 1 (show "Finish" label instead, clicking calls onExit)
    - Or: Next on last step calls onExit (research says "clamp to last step" but finishing is better UX)
    - Style with Tailwind: `flex items-center gap-3 px-4 py-2 bg-surface border-b` (fixed above graph, not inside SVG)
    - Narrative text gets `flex-1 text-sm` to fill available space
    - Responsive: on mobile (< 768px), stack vertically or truncate narrative
    - CRITICAL: TourBar renders OUTSIDE the SVG element, above the graph container, as a fixed UI element. It does NOT position relative to SVG nodes.
