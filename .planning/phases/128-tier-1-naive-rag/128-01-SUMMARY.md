- **Initial `uv pip install --dry-run` failed in sandbox** — Default `~/.cache/uv/sdists-v8/` is outside the sandbox-writable allowlist. Resolved by setting `UV_CACHE_DIR=$TMPDIR/uv-cache` for the dry-run command. Resolution was sub-second once cache redirected.
- **`uv pip install --dry-run --python python3.11` rejected** — Existing `.venv` is Python 3.13.1 (per `pyvenv.cfg`); plan's `--python python3.11` flag conflicted with the venv. Resolved by dropping the flag and letting uv use the existing venv (3.13.1, well within the project's `requires-python = ">=3.10"` constraint). The plan's intent (dry-run resolution gate) was preserved.
- **pymupdf not appearing in dry-run "would install" diff** — Initially looked like a missing dep, but `uv pip list` confirmed pymupdf 1.27.2.3 already present (installed earlier as part of `[curation]` extras during Phase 127). Forced `--reinstall` confirmed pymupdf is in the tier-1 resolution graph at a satisfying version. Verification gate is green.
