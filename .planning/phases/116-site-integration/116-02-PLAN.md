13. /guides/claude-code/agent-sdk/
14. /guides/claude-code/computer-use/
15. /guides/claude-code/cheatsheet/

For each page, run:
```bash
lighthouse "http://localhost:4321/guides/claude-code/$slug/" \
  --output json --quiet \
  --chrome-flags="--headless --no-sandbox" \
  --only-categories=performance,accessibility,best-practices,seo \
  | jq '{page: .finalUrl, perf: (.categories.performance.score * 100), a11y: (.categories.accessibility.score * 100), bp: (.categories["best-practices"].score * 100), seo: (.categories.seo.score * 100)}'
```

Collect results into a summary table. The threshold is 90+ for ALL four categories on ALL pages.

If any page scores below 90 in any category:
- Note the specific page and category
- Check if the issue is systemic (affects all pages) or page-specific
- If page-specific (e.g., cheatsheet with large SVGs), document the score and the likely cause
- Scores of 89 or above on a single category for a single page may be acceptable due to Lighthouse variability (scores fluctuate +/- 3-5 points between runs)

After auditing, kill the preview server process.
