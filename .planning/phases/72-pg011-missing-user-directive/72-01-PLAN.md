Create `src/lib/tools/dockerfile-analyzer/rules/security/PG011-missing-user-directive.ts` following the exact self-contained module pattern used by DL3002 and PG010.

**Rule metadata:**
- `id`: `'PG011'`
- `title`: `'Add a USER directive to avoid running as root'`
- `severity`: `'warning'` (matches DL3002 convention for USER-related issues)
- `category`: `'security'`
- `explanation`: Expert-level text referencing CIS Docker Benchmark 4.1, container escape risk, Kubernetes Pod Security Standards, and the invisible nature of missing USER vs explicit USER root. See the research file for the full recommended explanation text.
- `fix.description`: Instructs adding a non-root USER with explicit UID/GID for deterministic builds.
- `fix.beforeCode`: A simple Dockerfile with no USER (e.g., `FROM node:22-bookworm-slim\nWORKDIR /app\nCOPY . .\nCMD ["node", "server.js"]`)
- `fix.afterCode`: Same Dockerfile with `groupadd`/`useradd` and `USER appuser` added (see research for exact text).

**Detection logic (`check` method):**
1. Get all FROM instructions via `dockerfile.getFROMs()`
2. Return empty if `froms.length === 0`
3. Get last FROM via `froms.at(-1)` -- this is the final stage
4. Skip `FROM scratch`: if `lastFrom.getImageName() === 'scratch'`, return empty (no user system exists)
5. Get `lastFromLine` from `lastFrom.getRange().start.line`
6. Filter all instructions to find USER instructions in the final stage: `dockerfile.getInstructions().filter(inst => inst.getKeyword() === 'USER' && inst.getRange().start.line > lastFromLine)`
7. CRITICAL non-overlap boundary: if `userInstructions.length > 0`, return empty immediately (defer to DL3002 which handles explicit USER root/0)
8. If zero USER instructions in final stage, push a violation on the last FROM line: `line: range.start.line + 1` (1-based), `column: 1`
9. Message: `'No USER directive in the final stage. The container will run as root. Add a USER instruction to switch to a non-root user.'`

**Registry update in `src/lib/tools/dockerfile-analyzer/rules/index.ts`:**
1. Add import after PG010 line (line 17): `import { PG011 } from './security/PG011-missing-user-directive';`
2. Add `PG011,` after `PG010,` in the allRules array (after line 73)
3. Update security count comment from `(14)` to `(15)` in both locations (import section comment on line 3, and array comment on line 59)

Do NOT modify any other files -- engine.ts, scorer.ts, related.ts, and [code].astro all auto-adapt from the allRules array.
