  <name>Task 1: Create shared AST helpers for YAML traversal</name>
  <files>src/lib/tools/gha-validator/rules/security/ast-helpers.ts</files>
  <action>
Create `src/lib/tools/gha-validator/rules/security/ast-helpers.ts` with three shared traversal functions used across multiple security rules:

1. **resolveKey(node, key): any** -- Given a YAML Map node, find and return the value of a specific key. Returns null if node is not a Map or key not found. Uses `isMap`, `isPair`, `isScalar` from the `yaml` library.

2. **forEachUsesNode(ctx, callback)** -- Iterate over all `uses:` values in the workflow. Traverses: `jobs -> [jobName] -> steps -> [n] -> uses`. Callback signature: `(usesValue: string, node: any, jobName: string) => void`. Used by GA-C001, GA-C002, GA-C008. Implementation:
   - Get `jobs` node via resolveKey(doc.contents, 'jobs')
   - Guard: if not isMap, return
   - For each job pair: get jobName from key, get `steps` via resolveKey(value, 'steps')
   - Guard: if steps not isSeq, continue
   - For each step: if isMap, find `uses` pair, if value isScalar, call callback with String(value.value), the value node, and jobName
   - Skip `uses:` values starting with `docker://` or `./` (local/docker actions have no version to pin)

3. **forEachRunNode(ctx, callback)** -- Iterate over all `run:` values in the workflow. Traverses: `jobs -> [jobName] -> steps -> [n] -> run`. Callback signature: `(runValue: string, node: any, jobName: string, stepIndex: number) => void`. Used by GA-C005. Implementation similar to forEachUsesNode but for `run` key. The runValue is the full string content (handles multiline `run: |` blocks correctly via `String(node.value)`).

Import `isMap, isPair, isScalar, isSeq` from `yaml`. Import `GhaRuleContext` from `../../types`.
  </action>
