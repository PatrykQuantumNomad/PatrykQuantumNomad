Create 5 rule files:

**GA-C006-pull-request-target.ts:**
- Export `GAC006: GhaLintRule`
- id: 'GA-C006', severity: 'warning', category: 'security'
- explanation: Reference Shai Hulud v2 worm (Nov 2025). pull_request_target runs with write permissions in the base repo context.
- check(): Use resolveKey to find `on` key. If `on` is a Map, check for `pull_request_target` key. If found, check if the value is a Map with any of: `paths`, `paths-ignore`, `branches`, `branches-ignore`. If none present, emit violation. If `on` is a Seq (list of event names), check if it contains `pull_request_target` string (no filters possible in short form = always flag).
- fix: Add branch or path restrictions to pull_request_target

**GA-C007-hardcoded-secrets.ts:**
- Export `GAC007: GhaLintRule`
- id: 'GA-C007', severity: 'warning', category: 'security'
- SECRET_PATTERNS array from research: 9 patterns (ghp_, github_pat_, gho_, ghs_, ghr_, AKIA, sk-, xoxb-, SG.)
- SECRETS_REF_RE: `/^\$\{\{\s*secrets\./` to skip proper secret references
- check(): Traverse all scalar values in the document recursively. For each string value that is not a secrets reference, test against all SECRET_PATTERNS. If match, emit violation with the pattern name.
- Recursion: Use a `walkScalars(node, lineCounter, callback)` helper within this file that traverses Map items, Seq items, and Pair values to find all Scalar nodes.
- fix: Use `${{ secrets.MY_TOKEN }}` instead of hardcoded values

**GA-C008-third-party-no-sha.ts:**
- Export `GAC008: GhaLintRule`
- id: 'GA-C008', severity: 'warning', category: 'security'
- check(): Use forEachUsesNode. Skip if uses value owner starts with `actions/` or `github/` (first-party). For remaining (third-party), extract ref after `@`. If ref is NOT a 40-char hex SHA (`/^[0-9a-f]{40}$/i`), emit violation.
- Difference from GA-C001: C001 flags FIRST-party semver tags, C008 flags ANY third-party ref that isn't a SHA. C001 is about semver mutability, C008 is about untrusted supply chain.
- fix: Pin third-party action to full commit SHA

**GA-C009-dangerous-token-scopes.ts:**
- Export `GAC009: GhaLintRule`
- id: 'GA-C009', severity: 'warning', category: 'security'
- DANGEROUS_COMBOS: Array of `[scope1, scope2]` pairs:
  - `['contents', 'actions']` -- can modify and trigger workflows
  - `['packages', 'contents']` -- supply chain risk
  - `['id-token', '*']` -- id-token:write + any other write scope
- check(): Use resolveKey to find top-level `permissions`. If isMap, extract all scopes with `write` value. Check each DANGEROUS_COMBO: if both scopes have write permissions, emit violation. For the `id-token` + `*` combo, check if `id-token: write` exists alongside any other scope with `write`.
- Also check job-level permissions the same way.
- fix: Use minimal required permission scopes. Split into separate jobs if different scopes needed.

**GA-C010-self-hosted-runner.ts:**
- Export `GAC010: GhaLintRule`
- id: 'GA-C010', severity: 'info', category: 'security'
- check(): Traverse `jobs -> [jobName] -> runs-on`. If value isScalar and contains `self-hosted` (case-insensitive), emit violation. If value isSeq, check if any item isScalar with value `self-hosted`.
- Message: Informational reminder about self-hosted runner security (no network isolation, persistent state, credential exposure).
- fix: Consider using GitHub-hosted runners or ensure self-hosted runners follow security hardening guidelines
