    - Test: All positions have x within [0, 1200] and y within [0, 900] (within viewBox bounds)
    - Test: Every node ID in nodes.json has a corresponding position in layout.json
  </behavior>
  <action>
    1. Install d3-force and @types/d3-force: `npm install d3-force @types/d3-force`

    2. Create `src/lib/ai-landscape/layout-schema.ts`:
       - Import z from 'astro/zod'
       - Define layoutPositionSchema: { id: string, x: number, y: number }
       - Define layoutMetaSchema: { width: number, height: number, ticks: number, generated: string }
       - Define layoutSchema: { meta: layoutMetaSchema, positions: z.array(layoutPositionSchema) }
       - Export all schemas and inferred types: LayoutPosition, LayoutMeta, Layout

    3. Create `scripts/generate-layout.mjs`:
       - Read nodes.json and graph.json from src/data/ai-landscape/ using readFileSync
       - Define constants: WIDTH=1200, HEIGHT=900, TICKS=300, PAD=40
       - Define cluster centroids for spatial grouping (approximate positions):
         ai: {x: 420, y: 225}, ml: {x: 360, y: 360}, nn: {x: 300, y: 495},
         dl: {x: 360, y: 585}, genai: {x: 540, y: 675}, levels: {x: 900, y: 180},
         agentic: {x: 900, y: 495}, agent-frameworks: {x: 960, y: 585},
         devtools: {x: 840, y: 765}
       - Clone node data into simNodes (only id and cluster -- avoid d3-force mutating original data)
       - Set initial position of each simNode to its cluster centroid
       - Clone edge data into simLinks (only source and target)
       - Create forceSimulation with: forceLink (distance: 60, strength: 0.3, id accessor),
         forceManyBody (strength: -200, distanceMax: 300), forceCenter (WIDTH/2, HEIGHT/2),
         forceCollide (radius: 25), forceX (cluster centroid x, strength: 0.15),
         forceY (cluster centroid y, strength: 0.15)
       - Call simulation.stop() then tick 300 times in a for loop
       - After simulation: clamp all positions to [PAD, WIDTH-PAD] for x and [PAD, HEIGHT-PAD] for y
       - Round positions to 2 decimal places
       - Write layout.json with meta and positions to src/data/ai-landscape/layout.json
       - Log summary: node count, dimensions, generation time

    4. Update `package.json` prebuild script:
       - Chain: `"prebuild": "node scripts/download-actionlint-wasm.mjs && node scripts/generate-layout.mjs"`
       - Add standalone: `"generate-layout": "node scripts/generate-layout.mjs"`

    5. Run the script: `node scripts/generate-layout.mjs` to produce layout.json

    6. Write tests in `src/lib/ai-landscape/__tests__/layout-schema.test.ts` per the behavior block above.
       Import the real layout.json and nodes.json for integration assertions.

    IMPORTANT: d3-force mutates input arrays. Always clone data before passing to forceSimulation.
    IMPORTANT: Use `import { forceSimulation, forceLink, forceManyBody, forceCenter, forceCollide, forceX, forceY } from 'd3-force'` -- never the umbrella d3 package.
  </action>

  <action>
    Create `src/lib/ai-landscape/svg-builder.ts`:

    1. Import types: AiNode, Edge, Cluster from './schema' and LayoutPosition, LayoutMeta from './layout-schema'

    2. Define constants:
       - NODE_RADIUS = 18 for standard nodes
       - ROOT_NODE_RADIUS = 24 for cluster head nodes (nodes whose id matches their cluster id, e.g., 'artificial-intelligence' for 'ai' cluster)
       - LABEL_FONT_SIZE = 9
       - FONT_FAMILY = "'DM Sans', sans-serif"

    3. Export function buildLandscapeSvg(nodes, edges, positions, clusters, meta) -> string:

       a. Build lookup maps: posMap (id -> {x,y}), nodeMap (id -> AiNode), clusterMap (id -> Cluster)

       b. Determine "root" node IDs: nodes where parentId is null (these get ROOT_NODE_RADIUS)

       c. Generate CSS style block inside SVG:
          - For each cluster: `.ai-cluster-{id} { fill: {color}; stroke: {darkColor}; stroke-width: 1.5; }`
          - For each cluster dark mode: `html.dark .ai-cluster-{id} { fill: {darkColor}; stroke: {color}; }`
          - Edge styling: `.ai-edge { stroke: var(--color-border); }` and `html.dark .ai-edge { stroke: var(--color-border); }`
          - Text styling: `.ai-label { fill: var(--color-text-primary); font-family: ${FONT_FAMILY}; font-size: ${LABEL_FONT_SIZE}px; }`

       d. Generate edges SVG (rendered FIRST for correct z-order):
          - Wrap in `<g class="edges">`
          - For each edge: `<line>` from source position to target position
          - hierarchy edges: stroke-width 2, solid
          - includes edges: stroke-width 1.5, stroke-dasharray="4 3"
          - Other edges: stroke-width 1, opacity 0.4
          - All edges use class="ai-edge"

       e. Generate nodes SVG (rendered SECOND, on top of edges):
          - Wrap in `<g class="nodes">`
          - For each node with a position:
            - `<circle>` with appropriate radius (ROOT_NODE_RADIUS if parentId is null, NODE_RADIUS otherwise)
            - class="ai-cluster-{cluster}"
            - `<text>` label below circle, text-anchor="middle", class="ai-label"
            - Strip parenthetical from name: node.name.replace(/ *\(.*\)/, '')

       f. Wrap in `<svg>` with:
          - viewBox="0 0 {width} {height}"
          - xmlns="http://www.w3.org/2000/svg"
          - role="img"
          - aria-label="AI Landscape force-directed graph showing {nodeCount} concepts across {clusterCount} clusters"
          - style="width:100%;height:auto;max-width:{width}px"

    4. Write tests in `src/lib/ai-landscape/__tests__/svg-builder.test.ts` per behavior block.
       Tests should import real data (nodes.json, graph.json, layout.json) for integration-level assertions.

    IMPORTANT: Do NOT hardcode hex colors in SVG attributes -- use CSS classes for cluster fills. This ensures dark mode works via html.dark toggle.
    IMPORTANT: Use var(--color-text-primary) and var(--color-border) for text and edge colors -- these are the existing CSS custom properties in this project.
    IMPORTANT: Render edges group before nodes group in the SVG (SVG paints later elements on top).
