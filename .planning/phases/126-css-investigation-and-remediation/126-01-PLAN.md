```js
          (await import('rollup-plugin-visualizer')).visualizer({

          }),
          (await import('rollup-plugin-visualizer')).visualizer({
            emitFile: false,
            filename: `.planning/reports/css-visualizer-${new Date().toISOString().replace(/\D/g, '').slice(0, 13)}.json`,
            template: 'raw-data',
            gzipSize: true,
            brotliSize: true,
          }),
        ],
      }
    : {},
```

**CRITICAL — emitFile MUST be false.** Per 126-RESEARCH.md Pitfall 1,
`emitFile: true` routes through Rollup's asset-emission and typically
lands the file in `dist/_astro/stats.html`, polluting the deploy artefact
AND risking Phase 123 sitemap determinism. Using `emitFile: false`
forces the plugin to write directly to the `filename` path as a regular
filesystem write, bypassing Rollup's pipeline.

**CRITICAL — filename path MUST NOT start with `dist/`.** Verify post-
build that no `stats.html` or `css-visualizer-*` file appears in `dist/`:
`find dist -name 'stats*' -o -name 'css-visualizer*'` must return empty.

**CRITICAL — top-level await in ESM config.** Astro 5 + Node 22+ supports
top-level `await` in `astro.config.mjs`. The dynamic `import()` pattern
means the visualizer module ONLY loads when ANALYZE=1, not on every
config parse. If for any reason top-level await is rejected (e.g., old
Node), fall back to a top-of-file static `import { visualizer } from
'rollup-plugin-visualizer'` and gate only the plugins array — this still
loads the module but the plugins don't run.

4. For each CSS href, read the file from `dist/`, compute raw byte
   length, gzip length (`zlib.gzipSync`), brotli length

   blocks (`/@font-face\s*\{/g`).
5. Walk EVERY route's `index.html` under `dist/**/` (use readdirSync
   with `withFileTypes: true` + recursion; limit depth to 4; skip
   `_astro/`, `fonts/`, `open-graph/`, `wasm/`, any file not named
   `index.html`) and for each route record `{route, cssHrefs: [...]}`.
   This produces the route→chunks map that proves "shared vs per-route"
   classification (RESEARCH.md Pitfall 3).
6. Compute top-level summary: total unique CSS chunks, total raw bytes,
   total gzipped bytes, count of routes each chunk appears on, which

   <5 routes). Include a `homepageChunks` array listing ONLY the chunks

8. Print a single-line stdout summary:
   `[diagnose-homepage-css] homepage loads N chunks / totalRaw bytes / totalGzip bytes — report: <path>`.
