**The FOUR invariants the verifier asserts:**

**Invariant 1 — Homepage chunk count stable:**
Parse `dist/index.html`, extract all
`<link rel="stylesheet" href="/_astro/*.css">` hrefs. Assert the count
equals `baseline.summary.homepageChunkCount` exactly. Fail with hint
"homepage now loads N chunks (was M); a new CSS chunk was added or
removed. If intentional, update the baseline via
`node scripts/diagnose-homepage-css.mjs` and `git add -f` the new
report."

**Invariant 2 — Per-chunk raw byte ceiling:**
For each chunk in `baseline.homepageChunks`, find the equivalent chunk
in the current build by matching the file name BEFORE the hash suffix
(e.g., `about.*.css`, `_slug_.*.css`). Compute raw byte length of the
current file. Assert `current.rawBytes <= baseline.rawBytes * HEADROOM`
where HEADROOM = 1.02 (2% tolerance — small enough to catch 5KB+ bloat
but lenient for de-minimis Tailwind churn). Fail with hint identifying
which chunk grew and by how many bytes.

**Invariant 3 — Homepage total raw byte ceiling:**
Sum raw bytes of all homepage chunks. Assert
`total <= baseline.summary.homepageRawBytes * TOTAL_HEADROOM` where
TOTAL_HEADROOM = 1.05 (5% — absorbs normal content-driven growth).
Fail with a breakdown showing baseline vs current per chunk.

**Invariant 4 — Homepage total gzip byte ceiling:**
Same as Invariant 3 but on gzipped bytes (using `zlib.gzipSync`).
Ceiling = `baseline.summary.homepageGzipBytes * 1.05`. Fail with a
per-chunk gzip breakdown.

**Chunk-name matching helper (robust against hash suffix changes):**

```js
function stableName(href) {
  // /_astro/about.C49NBCVn.css -> about.css
  // /_astro/_slug_.CIgCJX9d.css -> _slug_.css
  const base = href.split('/').pop();   // e.g., 'about.C49NBCVn.css'
  return base.replace(/\.[A-Za-z0-9_-]{8}\.css$/, '.css');
}
```
