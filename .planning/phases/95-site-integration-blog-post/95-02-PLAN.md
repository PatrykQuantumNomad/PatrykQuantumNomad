Content structure (aim for 2500+ words, pure MDX prose with no component imports):

**Opening thesis (3-4 paragraphs):** The unique angle is "the context window is the product." Most developers treat Claude Code as a fancy chat box. They type a request, get output, and move on. The developers who get 10x results treat the context window as a product they actively design: CLAUDE.md files, skill definitions, hook automations, and MCP integrations are all products designed to shape what the agent sees and how it behaves. The context window is not a limitation to work around -- it is the primary interface you control.

This framing positions context management as the hidden skill of AI-assisted development. It naturally leads into the chapter summaries because each chapter addresses a different aspect of context: what the agent knows (CLAUDE.md), what it costs (models), what it can do (skills, hooks, MCP), and how it scales (worktrees, agent teams).

Link to the guide landing page: `[Claude Code Guide](/guides/claude-code/)`.
