## Production Follow-Up

The 5 fix-forward commits that added inline @font-face priming (fix 3), inline font-family activation (fix 4), and style-before-preload source ordering (fix 5) were motivated by preload-unused warnings observed under `npx astro preview` (Node.js HTTP/1.1, localhost, single-threaded). GitHub Pages serves over HTTP/2 CDN with parallel pipelining and edge caching, which typically delivers per-route CSS chunks substantially faster than the Node preview server. The inline priming patches may be over-engineering for the production delivery path — the preload-usage window may never close early there.

**Recommendation:** Do not roll back the inline priming. It is defensive, adds under 2 KB of extra HTML, and guarantees first-paint font activation regardless of delivery path. After merge to main and deployment to the `patrykgolabek.dev` GitHub Pages site, re-verify with Chrome DevTools Network (Font filter) over a 10-route sample. If no preload-unused warnings appear in production, the belt-and-suspenders setup is correct steady state. If warnings do appear, the inline priming is doing necessary work. Either outcome is acceptable — this follow-up is informational, not a blocker.
