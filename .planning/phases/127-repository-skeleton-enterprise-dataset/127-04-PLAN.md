   ```
   cd ../rag-architecture-patterns

     --seeds scripts/seed_papers.json \
     --output dataset/papers/ \
     --output-manifest dataset/manifests/papers.json \
     --hops 2 \
     --max-papers 100 \
     2>&1 | tee /tmp/curate_realrun.log
   ```
   This will:
   - Take 8-15 min (rate-limited)
   - Download ~100 PDFs
   - Build dataset/manifests/papers.json with one entry per downloaded paper
   - Be idempotent: re-running skips PDFs already on disk

2. Verify all PDFs are LFS-tracked BEFORE staging (per Pitfall 1 — verify, don't assume):
   ```
   cd ../rag-architecture-patterns
   for pdf in dataset/papers/*.pdf; do
     attr=$(git check-attr filter "$pdf" | awk -F': ' '{print $NF}')
     if [ "$attr" != "lfs" ]; then
       echo "FAIL: $pdf has filter=$attr (expected lfs)"
       exit 1
     fi
   done
   echo "All PDFs LFS-tracked"
   ```

3. Verify manifest integrity:
   ```
   python -c "
   import json
   from pathlib import Path
   manifest = json.load(open('dataset/manifests/papers.json'))
   print(f'Manifest entries: {len(manifest)}')
   missing = [m['filename'] for m in manifest if not (Path('dataset/papers') / m['filename']).exists()]
   assert not missing, f'manifest references missing files: {missing[:5]}'
   extras = set(p.name for p in Path('dataset/papers').glob('*.pdf')) - set(m['filename'] for m in manifest)
   assert not extras, f'PDFs without manifest entries: {sorted(extras)[:5]}'
   print('Manifest <-> filesystem consistent')
   "
   ```

4. Check total LFS payload size (must be <500 MB per the budget):
   ```
   du -sh dataset/papers/
   du -sh dataset/manifests/
   ```
   If >500 MB, this is a flag — but proceed (curator approved in Task 2). Document in SUMMARY.

5. Stage and commit. The commit triggers LFS upload to GitHub:
   ```
   git add dataset/papers/ dataset/manifests/papers.json
   git status                                # Sanity: should show *.pdf in LFS, not in regular blobs
   git lfs ls-files | wc -l                  # Should match number of PDFs
   git commit -m "feat: download ~100 arXiv PDFs forming RAG citation cluster (REPO-02)"
   ```

6. Push to origin. This is the LFS bandwidth-consuming step:

   git push origin main
