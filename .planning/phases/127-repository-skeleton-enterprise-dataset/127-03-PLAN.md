Write `scripts/extract_figures.py` per 127-RESEARCH.md Pattern 3:

Required CLI:
```
--pdf PATH               # Single-PDF mode
--all DIR                # Batch mode: process every *.pdf in DIR
--out DIR                # Default: dataset/images/
--manifest PATH          # Default: dataset/manifests/figures.json
--min-dim N              # Default: 200 (per Pitfall 7)

--dry-run                # Print what WOULD be extracted, write nothing
```

Required behavior:
- Use `import fitz` (PyMuPDF; the `fitz` module name is correct — pymupdf >= 1.24 also exposes `import pymupdf` but `fitz` is canonical per docs).
- For each PDF:
  1. Derive `paper_id` from the filename's arxiv short id prefix (e.g., `2005.11401_lewis_rag.pdf` → `2005.11401`). Be defensive: if regex `^(\d{4}\.\d{4,5})` doesn't match, log a warning and skip.
  2. `doc = fitz.open(pdf_path)`.
  3. For each page (1-indexed for manifest):
     - For each `img` in `page.get_images(full=True)`:
       - `xref = img[0]`
       - Apply xref dedup within this PDF: maintain a set of seen xrefs; skip if already exported (per Pitfall 7 — same logo on every page reuses xref).
       - `rects = page.get_image_rects(xref)`; if empty, skip.
       - `bbox = rects[0]` (first occurrence on page).
       - `base = doc.extract_image(xref)`; ext = `base["ext"]` (png/jpg/etc.); data = `base["image"]`.
       - Apply size filter: load via `PIL.Image.open(BytesIO(data))` to get true width/height (PyMuPDF's bbox is page coords, not pixel coords). Skip if `width < min_dim or height < min_dim`.
       - Apply aspect filter: skip if `max(w/h, h/w) > max_aspect_ratio`.
       - Save as `{paper_id}_fig_{counter:03d}.{ext}` under `--out`.
       - Append manifest entry per the shape in <interfaces>: figure_id, paper_id, page_number, bbox (4-float list), caption ("" — manual fill later), filename, size_bytes, width, height.
- DO NOT attempt automatic caption extraction — caption is a manual field per Open Question #4 in research.
- On `--dry-run`: print "Would extract N figures from M PDFs" with the post-filter counts; do not write image files or manifest.
- Idempotent: if `manifest` already exists, load it and dedupe by `figure_id` before appending. If a figure file already exists at the destination, skip (allows resume).
- Use rich.Progress for batch mode.
- Note in code comments: "Per Pitfall 7 in 127-RESEARCH.md — heuristic filtering catches ~90%+; remaining garbage requires manual review in Plan 05 before manifest commit."
