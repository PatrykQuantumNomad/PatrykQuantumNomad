<objective>
Implement sharing capabilities: score badge PNG download, URL hash state encoding with `#gha=` prefix, and 3-tier share fallback. Wire all sharing features into the editor and results panels.

Purpose: Sharing enables users to distribute their validation results via direct links, social media badges, and native share dialogs. URL hash encoding with lz-string (already installed) allows one-click sharing of analyzed workflows. The badge PNG provides a visual score indicator for READMEs and social media.

Output: Three sharing utility modules, URL state test suite, and updated editor/results panels with share/download buttons.
</objective>

  <behavior>
    URL state tests (url-state.test.ts):
    - encodeGhaState returns string starting with '#gha='
    - decodeGhaState('#gha=...') returns original YAML (roundtrip)
    - decodeGhaState('#other=...') returns null (wrong prefix)
    - decodeGhaState('') returns null (empty string)
    - decodeGhaState('#gha=') returns empty string or null (empty content)
    - Roundtrip works with multiline YAML (10+ lines)
    - Compressed output length is shorter than input for typical YAML
  </behavior>
  <action>
1. **url-state.ts**: Create `encodeGhaState(yaml: string): string` and `decodeGhaState(hash: string): string | null`.
   - Use `lz-string` imports: `compressToEncodedURIComponent` and `decompressFromEncodedURIComponent`
   - Import as `import LZString from 'lz-string'` and use `LZString.compressToEncodedURIComponent()` / `LZString.decompressFromEncodedURIComponent()`
   - `encodeGhaState`: return `#gha=${compressToEncodedURIComponent(yaml)}`
   - `decodeGhaState`: check hash starts with `#gha=`, extract substring after `#gha=`, decompress. Return null if prefix mismatch or decompression fails.
   - Export a `GHA_URL_SOFT_LIMIT = 6000` constant. Export a `isUrlTooLong(hash: string): boolean` helper that checks if the hash exceeds the soft limit.

2. **badge-png.ts**: Create `downloadScoreBadge(score: number, grade: string): void`.
   - Create off-screen canvas at 400x200 (2x DPR for crisp rendering = 800x400 actual pixels)
   - Draw dark background (#1a1b26 matching site theme)
   - Draw score arc (replicate ScoreGauge logic): circle at left side, arc length proportional to score/100
   - Use `getGradeColor()` logic (copy from ScoreGauge.tsx): A=green, B=lime, C=yellow, D=orange, F=red
   - Draw score number centered in arc, grade letter below
   - Draw "GHA Validator Score" text on right side
   - Use `canvas.toBlob()` to create PNG, trigger download via anchor element with `download` attribute
   - Filename: `gha-score-${score}-${grade}.png`

3. **share-fallback.ts**: Create `shareUrl(url: string, title: string): Promise<'shared' | 'copied' | 'prompted'>`.
   - Tier 1: If `navigator.share` exists, try `navigator.share({ title, url })`. Catch `AbortError` (user cancel) as no-op, return 'shared'.
   - Tier 2: If `navigator.clipboard?.writeText` exists, copy URL, return 'copied'.
   - Tier 3: Fall back to `window.prompt('Copy this link to share:', url)`, return 'prompted'.
   - Return value indicates which tier was used (for caller to show appropriate toast/feedback).

4. **Write tests first** (TDD): Create url-state.test.ts with all behavior cases above. Run to see RED. Then implement url-state.ts to turn GREEN. badge-png.ts and share-fallback.ts are browser-only (Canvas/navigator APIs) so tested via build verification, not unit tests.

1. **GhaEditorPanel.tsx** -- URL hash read on mount and write after analysis:

   a. Import `decodeGhaState` and `encodeGhaState` from `../../lib/tools/gha-validator/share/url-state`.

   b. **Read hash on mount:** Add a `useEffect` that runs ONCE after the editor is mounted (depends on the EditorView ref being available). Logic:
      - Read `window.location.hash`
      - Call `decodeGhaState(hash)`
      - If non-null, replace the editor content with the decoded YAML using `view.dispatch({ changes: { from: 0, to: view.state.doc.length, insert: decodedYaml } })`
      - Then auto-trigger analysis (call the same analyze function used by the Analyze button)
      - This ensures shared links load the encoded workflow and immediately show results

   c. **Write hash after analysis:** After a successful analysis (after Pass 1 results are set), update the URL hash with the current editor content:
      - `window.history.replaceState(null, '', encodeGhaState(currentYaml))`
      - Use `replaceState` (not `pushState`) to avoid polluting browser history
      - Only update hash when analysis is triggered by user action (not on initial hash-decode load to prevent infinite loop). Use a ref flag `isHashLoad` to distinguish.

2. **GhaResultsPanel.tsx** -- Add Share and Download Badge buttons:

   a. Import `downloadScoreBadge` from `../../lib/tools/gha-validator/share/badge-png`.
   b. Import `shareUrl` from `../../lib/tools/gha-validator/share/share-fallback`.
   c. Import `useState` for toast feedback.

   d. **Add button row** below the ScoreGauge (inside the results tab, visible when `result` exists):
      - "Download Badge" button: calls `downloadScoreBadge(result.score.overall, result.score.grade)` on click. Style as a secondary/ghost button matching existing UI patterns.
      - "Share Link" button: calls `shareUrl(window.location.href, 'GitHub Actions Validator Score')`. On 'copied' result, show brief "Copied!" text feedback (use local state with 2-second timeout). Style matching the Download button.

   e. **Button styling:** Use existing Tailwind patterns from the codebase. Buttons should be small, inline, and not visually dominate the score gauge. Use `text-sm`, `px-3 py-1.5`, `rounded-lg`, `border border-[var(--color-border)]`, `hover:border-[var(--color-accent)]`, `transition-colors`.

   g. **URL length warning:** Import `isUrlTooLong` from url-state. After `shareUrl` is called, if the current URL hash is too long, log a console warning but still allow sharing. Do not block the user.
