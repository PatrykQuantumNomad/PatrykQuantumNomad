Props interface (inside Astro frontmatter fence):
- stat: string (required) -- the big number, e.g. "4x", "17%", "less than 10%"
- label: string (required) -- description text below the number
- source: string (optional) -- citation source text

Destructure: const { stat, label, source } = Astro.props;

HTML structure:
- Root element: `<aside>` with `class="not-prose my-8 py-6 text-center"` and `data-stat-highlight` attribute
- Stat number: `<p>` with `class="font-heading font-bold text-4xl sm:text-5xl text-[var(--color-accent)] mb-2"` -- the large, accent-colored number creates visual dominance
- Label: `<p>` with `class="text-[var(--color-text-primary)] text-sm font-medium uppercase tracking-wider"` -- uppercase small text for scannable label
- Source (conditional): render only when source prop is truthy. `<p>` with `class="text-[var(--color-text-secondary)] text-xs mt-2 italic"` -- de-emphasized citation

Key rules:
- NO hardcoded hex colors -- use only var(--color-*) CSS custom properties
- NO style blocks, NO script blocks
- Use `not-prose` on root element to escape typography plugin
- Use `<aside>` semantic element (this is supplementary content)

The research recommends a minimal borderless style (no background, no border) where the large stat number itself creates the visual distinction through typography scale. This is lighter than KeyTakeaway's boxed style and more appropriate for repeated use (the essay uses 6+ StatHighlight instances).

- Definition body: `<div>` with `class="text-[var(--color-text-primary)] text-base leading-relaxed [&>p]:mb-2 [&>p:last-child]:mb-0"` containing `<slot />` for MDX content

Key rules:

- Use `<h3>` (not `<h2>`) to avoid interfering with blog TOC which uses `.prose h2` selectors
- The `[&>p]:mb-2 [&>p:last-child]:mb-0` classes handle paragraph spacing within slotted MDX content
