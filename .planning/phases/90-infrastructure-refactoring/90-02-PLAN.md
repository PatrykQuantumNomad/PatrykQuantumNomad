    Create `src/components/guide/CodeBlock.astro` — a component that renders syntax-highlighted code blocks with a file-path tab header and NO GitHub source link.

    Per user locked decisions:
    - Filename tab-style header showing the file path (mimics editor tab, subtle background)
    - Copy-to-clipboard button always visible (top-right corner) — provided by astro-expressive-code automatically
    - Optional caption field below the code block for brief explanations
    - No GitHub source link

    Implementation:
    ```astro
    ---
    import { Code } from 'astro-expressive-code/components';

    interface Props {
      /** The raw source code to display */
      code: string;
      /** Language identifier for syntax highlighting (e.g. "typescript", "bash") */
      lang: string;
      /** Optional file path displayed as tab-style header */
      title?: string;
      /** Optional caption displayed below the code block */
      caption?: string;
    }

    const { code, lang, title, caption } = Astro.props;
    ---

    <div class="my-6 not-prose">
      {title && (
        <div class="flex items-center px-3 py-2 bg-[var(--color-surface-alt)] border border-b-0 border-[var(--color-border)] rounded-t-lg text-sm">
          <span class="font-mono text-[var(--color-text-secondary)] truncate">{title}</span>
        </div>
      )}
      <div class:list={[title && '[&_.expressive-code]:!rounded-t-none [&_.expressive-code]:!mt-0 [&_.expressive-code]:!border-t-0']}>
        <Code code={code} lang={lang} title={title} />
      </div>
      {caption && (
        <p class="text-xs text-[var(--color-text-secondary)] mt-2 italic">{caption}</p>
      )}
    </div>
    ```

    Key differences from CodeFromRepo:
    - No `filePath`, `templateRepo`, `versionTag` props (no source attribution)
    - No "View source" link with external link icon
    - No import of `buildGitHubFileUrl`
    - Simpler props: just code, lang, title, caption
    - The header bar only shows the file path text (no link on the right side)
