Create `src/components/ai-landscape/MiniMap.tsx` — a read-only mini-map component that shows a scaled-down view of the full graph with a viewport rectangle.

**Props interface:**
```typescript
import React from 'react';
import type { ZoomTransform } from 'd3-zoom';
import type { LayoutPosition, LayoutMeta } from '../../lib/ai-landscape/layout-schema';
import type { Cluster } from '../../lib/ai-landscape/schema';

interface MiniMapProps {
  positions: LayoutPosition[];
  clusters: Cluster[];
  clusterMap: Map<string, Cluster>;
  nodeClusterMap: Map<string, string>;  // nodeId -> clusterId
  transform: ZoomTransform;
  meta: LayoutMeta;
}
```

**Component implementation:**
```typescript
export const MiniMap = React.memo(function MiniMap({
  positions,
  clusters,
  clusterMap,
  nodeClusterMap,
  transform,
  meta,
}: MiniMapProps) {
  const MINI_WIDTH = 160;
  const MINI_HEIGHT = MINI_WIDTH * (meta.height / meta.width);

  // Viewport rectangle: invert the d3-zoom transform
  // to find what portion of the graph is visible
  const viewX = -transform.x / transform.k;
  const viewY = -transform.y / transform.k;
  const viewW = meta.width / transform.k;
  const viewH = meta.height / transform.k;

  return (
    <div
      className="absolute bottom-3 right-3 rounded-lg border border-[var(--color-border)] bg-[var(--color-surface)] shadow-sm overflow-hidden"
      style={{ width: MINI_WIDTH, height: MINI_HEIGHT }}
      aria-hidden="true"
    >
      <svg
        width={MINI_WIDTH}
        height={MINI_HEIGHT}
        viewBox={`0 0 ${meta.width} ${meta.height}`}
        className="block"
      >
        {/* Node dots colored by cluster */}
        {positions.map((pos) => {
          const clusterId = nodeClusterMap.get(pos.id);
          const cluster = clusterId ? clusterMap.get(clusterId) : undefined;
          return (
            <circle
              key={pos.id}
              cx={pos.x}
              cy={pos.y}
              r={8}
              fill={cluster?.color ?? 'var(--color-text-secondary)'}
              opacity={0.6}
            />
          );
        })}
        {/* Viewport rectangle */}
        <rect
          x={viewX}
          y={viewY}
          width={viewW}
          height={viewH}
          fill="var(--color-accent)"
          fillOpacity={0.08}
          stroke="var(--color-accent)"
          strokeWidth={6}
          rx={4}
        />
      </svg>
    </div>
  );
});
```

Key design decisions:
- `React.memo` wrapping prevents unnecessary re-renders when only unrelated state changes (the transform changes frequently during pan/zoom, but the component is cheap to render — memo is defensive for future growth)
- `aria-hidden="true"` — the mini-map is a visual orientation aid, not interactive; screen readers skip it
- Node dots use r=8 (visible at mini scale but not too large) with cluster color for orientation
- Viewport rectangle has a subtle fill (8% opacity) plus a solid stroke for visibility
- The component is pure derivation — no new state, no side effects
- `nodeClusterMap` is passed in (computed once in parent via useMemo) rather than requiring full AiNode[] — keeps props lightweight
- Read-only: no click-to-pan (per research recommendation — start simple, add interactivity later if needed)
- Dark mode: cluster colors from the `color` field work in both themes since they're bright enough. For dark mode refinement, the parent can pass darkColor-aware map in the future.

**Note on dark mode cluster colors:** The mini-map uses `cluster.color` (light mode color). To support dark mode properly, add a CSS class approach or use an inline style that checks dark mode. For simplicity in this first implementation, use the light color — the mini-map background (surface) provides enough contrast in both modes. If dark mode colors are needed, the parent can precompute a map using a media query or CSS variables.
