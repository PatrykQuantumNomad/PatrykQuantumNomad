  <action>
Execute the following sequence as a single staged operation (all changes committed together as one atomic commit):

1. Create target directories:
   ```
   mkdir -p skills
   mkdir -p public/benchmarks
   ```

2. Move 4 skill directories from `public/skills/` to `skills/` at repo root using `git mv` (preserves file modes and rename history):
   ```
   git mv public/skills/compose-validator skills/
   git mv public/skills/dockerfile-analyzer skills/
   git mv public/skills/gha-validator skills/
   git mv public/skills/k8s-analyzer skills/
   ```

3. Move benchmark artifacts to `public/benchmarks/` using `git mv`:
   ```
   git mv public/skills/benchmark-all-skills.json public/benchmarks/
   git mv public/skills/benchmark-all-skills.md public/benchmarks/
   git mv public/skills/grading-summary.json public/benchmarks/
   git mv public/skills/compose-validator-workspace public/benchmarks/
   git mv public/skills/dockerfile-analyzer-workspace public/benchmarks/
   git mv public/skills/gha-validator-workspace public/benchmarks/
   git mv public/skills/k8s-analyzer-workspace public/benchmarks/
   ```

4. After all `git mv` operations, `public/skills/` should be empty. If the directory still exists, it will be gone from the index. Remove it from the working tree if needed so the symlink can be created at that path.

5. Create the symlink bridge (relative path, portable across clones):
   ```
   ln -s ../skills public/skills
   git add public/skills
   ```

6. Remove the stale `--help/` directory at repo root. It is untracked and contains a template SKILL.md. Use special quoting to avoid shell flag interpretation:
   ```
   rm -rf './--help'
   ```

7. Verify pre-commit state:
   - `ls skills/` shows exactly 4 directories: compose-validator, dockerfile-analyzer, gha-validator, k8s-analyzer
   - `readlink public/skills` returns `../skills`
   - `git ls-files -s 'skills/*/hooks/*.sh'` shows all 4 scripts with 100755 mode
   - `test ! -d './--help'` confirms stale directory is gone
   - `ls public/benchmarks/` shows 3 JSON/MD files and 4 workspace directories

8. Commit all staged changes as a single atomic commit. The commit message should indicate this is the directory restructure for skills.sh discovery.

IMPORTANT: Do NOT update `scripts/test-validator-hooks.sh` or any other script references to `public/skills/` in this task. The symlink bridge ensures those paths resolve correctly. Reference updates are deferred to Phase 84 (DOC-05).
  </action>
