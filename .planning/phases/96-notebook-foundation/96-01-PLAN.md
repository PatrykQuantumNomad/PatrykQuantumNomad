  <action>
Create `src/lib/eda/notebooks/types.ts` with complete nbformat v4.5 TypeScript interfaces. These MUST use snake_case property names to match the JSON output format (not camelCase).

Required types (all named exports):
- `MultilineString` = `string | string[]` (nbformat multiline_string type)
- `CellId` = `string` (pattern: `^[a-zA-Z0-9-_]+$`, 1-64 chars)
- `MarkdownCell` interface with: `id: CellId`, `cell_type: 'markdown'`, `metadata: Record<string, unknown>`, `source: MultilineString`, optional `attachments`
- `CodeCell` interface with: `id: CellId`, `cell_type: 'code'`, `metadata: Record<string, unknown>`, `source: MultilineString`, `outputs: Output[]`, `execution_count: number | null`
- `RawCell` interface with: `id: CellId`, `cell_type: 'raw'`, `metadata: Record<string, unknown>`, `source: MultilineString`
- `Cell` = `MarkdownCell | CodeCell | RawCell` discriminated union
- Output types: `ExecuteResult` (output_type: 'execute_result', data, metadata, execution_count), `DisplayData` (output_type: 'display_data', data, metadata), `StreamOutput` (output_type: 'stream', name: 'stdout'|'stderr', text), `ErrorOutput` (output_type: 'error', ename, evalue, traceback: string[])
- `Output` = union of all 4 output types
- `KernelSpec` interface: name, display_name, optional language
- `LanguageInfo` interface: name, optional version/codemirror_mode/file_extension/mimetype/pygments_lexer
- `NotebookMetadata` interface: kernelspec, language_info, index signature for extras
- `NotebookV4` interface: `nbformat: 4` (literal), `nbformat_minor: 5` (literal), metadata, cells: Cell[]

Use the exact interface definitions from the research RESEARCH.md Pattern 1 section. Add a file-level comment referencing the nbformat v4.5 JSON schema source URL.

Follow project conventions: named exports, 2-space indent, single quotes, no barrel file.
