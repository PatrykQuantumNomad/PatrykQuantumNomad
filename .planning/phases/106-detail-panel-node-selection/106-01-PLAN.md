**schema.ts — add groupRelationshipsByType function:**

Add a new exported function `groupRelationshipsByType` below the existing `getNodeRelationships` function. It takes `(nodeId: string, edges: Edge[], nodeMap: Map<string, {name: string; slug: string}>)` and returns an object with five arrays:

```ts
interface RelationshipGroup {
  label: string;           // "Part of", "Includes", "Enables", "Examples", "Related"
  items: { slug: string; name: string; edgeLabel: string }[];
}
```

Return type: `RelationshipGroup[]` (only groups with non-empty items).

Mapping logic (derive from edges where source or target matches nodeId):
- **"Part of"**: edges where `type` is `hierarchy` or `includes` AND `nodeId` is the TARGET (meaning the node is a child/part of the source). Item = the SOURCE node.
- **"Includes"**: edges where `type` is `hierarchy` or `includes` AND `nodeId` is the SOURCE (meaning the node contains the target). Item = the TARGET node.
- **"Enables"**: edges where `type` is `enables` or `applies`. Item = the OTHER node (not nodeId).
- **"Examples"**: edges where `type` is `example`. Item = the OTHER node (not nodeId).
- **"Related"**: edges where `type` is `relates`, `progression`, `characterizes`, `aspires`, or `standardizes`. Item = the OTHER node (not nodeId).

For each matched edge, resolve the other node's name and slug from nodeMap. Skip edges where the other node is not found in nodeMap. The `edgeLabel` is the edge's `label` field.

Also export the `RelationshipGroup` interface.

**useMediaQuery.ts — create React hook:**

Create a `useMediaQuery(query: string): boolean` hook. Implementation:
1. Initialize state with `() => typeof window !== 'undefined' ? window.matchMedia(query).matches : false` (SSR-safe lazy initializer)
2. In useEffect, create `MediaQueryList` via `window.matchMedia(query)`, add `change` event listener that updates state, return cleanup
3. Export as named export `useMediaQuery`

Use `useState` + `useEffect` from React. No external dependencies.

**Props interface (export as `DetailPanelProps`):**
```ts
interface DetailPanelProps {
  node: AiNode;
  edges: Edge[];
  nodeMap: Map<string, AiNode>;
  onClose: () => void;
  onShowAncestry: (nodeSlug: string) => void;
}
```

**Internal state:**
- `isSimple: boolean` — useState(true) for ELI5 toggle. `true` = simple description (default), `false` = technical.

**Rendered content (top to bottom):**

1. **Header row**: Node name as `<h2>` (text-lg font-heading font-bold) + close button (X icon, calls onClose). Use a simple SVG X or the text "×" entity.

2. **ELI5 toggle**: A small toggle/switch row with labels "Simple" and "Technical". Use a button with `role="switch"` and `aria-checked={isSimple}`. Styling: pill-shaped toggle, active side gets accent color. Default: simple (left/active).

3. **Description**: Show `node.simpleDescription` when isSimple is true, `node.technicalDescription` when false. Wrap in a `<p>` with `text-sm leading-relaxed text-[var(--color-text-secondary)]`.

4. **"How did we get here?" button**: A text button styled like a link (`text-xs text-[var(--color-accent)] hover:underline`). On click, calls `onShowAncestry(node.slug)`. Only render if the node has a parentId (non-null).

6. **Full page link**: At the bottom, a link to `conceptUrl(node.slug)` styled as a small button/pill: "View full page →". Use `text-xs font-mono` styling.
