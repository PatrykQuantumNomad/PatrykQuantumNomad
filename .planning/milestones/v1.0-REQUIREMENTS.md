## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Site Structure (5/5)

- [x] **SITE-01**: User sees a responsive 5-page site (Home, Blog, Blog Posts, Projects, About) that works on mobile, tablet, and desktop
- [x] **SITE-02**: User can navigate between all pages via a persistent header with 5-7 nav items
- [x] **SITE-03**: User sees an animated hero section on the home page with typing effect for role title and CTA buttons
- [x] **SITE-04**: User can find contact information (mailto link + LinkedIn) on the site
- [x] **SITE-05**: User can browse all 19 GitHub repos on the Projects page, grouped by category with descriptions and links

### Content & Blog (7/7)

- [x] **BLOG-01**: User can browse a list of blog posts sorted by date on /blog/
- [x] **BLOG-02**: User can read individual blog posts at /blog/[slug]/ with proper typography
- [x] **BLOG-03**: User sees syntax-highlighted code blocks with copy buttons in blog posts
- [x] **BLOG-04**: User sees estimated reading time on each blog post
- [x] **BLOG-05**: Draft posts are excluded from production builds
- [x] **BLOG-06**: User can filter blog posts by tag on /blog/tags/[tag]/ pages
- [x] **BLOG-07**: User sees an auto-generated table of contents from headings on blog posts

### Theme & Visual Effects (10/10)

- [x] **THEME-01**: User can toggle between dark and light mode, with preference persisted across sessions
- [x] **THEME-02**: Site defaults to user's OS color scheme preference
- [x] **THEME-03**: No flash of unstyled content (FOUC) on page load or View Transition navigation
- [x] **THEME-04**: User sees the "Quantum Explorer" particle canvas on the home page hero section
- [x] **THEME-05**: Particle effects pause when tab is hidden and reduce on mobile devices
- [x] **THEME-06**: Users with prefers-reduced-motion see a static fallback instead of animations
- [x] **THEME-07**: User experiences smooth view transitions between pages via ClientRouter
- [x] **THEME-08**: User sees scroll-triggered reveal animations on page sections
- [x] **THEME-09**: Site uses futuristic typography (Space Grotesk headings, Inter body, JetBrains Mono code)
- [x] **THEME-10**: Site meets WCAG 2.1 AA accessibility standards (semantic HTML, keyboard nav, contrast ratios)

### SEO & Discoverability (10/10)

- [x] **SEO-01**: Every page has unique title (50-60 chars) and meta description (150-160 chars)
- [x] **SEO-02**: Every page has canonical URL, Open Graph tags, and Twitter Card tags
- [x] **SEO-03**: Sitemap auto-generated at /sitemap-index.xml
- [x] **SEO-04**: RSS feed available at /rss.xml
- [x] **SEO-05**: robots.txt points to sitemap-index.xml
- [x] **SEO-06**: JSON-LD structured data on homepage (Person schema) and blog posts (BlogPosting schema)
- [x] **SEO-07**: Dynamic OG images auto-generated per blog post using Satori + Sharp
- [x] **SEO-08**: LLMs.txt file served for LLM-friendly content discovery
- [x] **SEO-09**: Site optimized for Generative Search Optimization (GEO) with structured, authoritative content
- [x] **SEO-10**: SEO keywords naturally integrated throughout site content

### Deployment & Infrastructure (4/4)

- [x] **INFRA-01**: Site deploys automatically via GitHub Actions using withastro/action@v5 on push to main
- [x] **INFRA-02**: Custom domain patrykgolabek.dev configured with CNAME in public/
- [x] **INFRA-03**: Site achieves Lighthouse 90+ scores across all categories on mobile
- [x] **INFRA-04**: astro.config.mjs configured with site: 'https://patrykgolabek.dev' and output: 'static'
