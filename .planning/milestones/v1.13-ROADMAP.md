# Milestone v1.13: GitHub Actions Workflow Validator

**Status:** ✅ SHIPPED 2026-03-04

## Overview

Browser-based GitHub Actions workflow validator with two-pass linting (SchemaStore JSON Schema + actionlint WASM), full workflow graph visualization, category-weighted scoring, and per-rule SEO documentation pages. First WASM-based tool in the portfolio suite.

## Milestone Summary

**Key Decisions:**
- Classic Worker with importScripts for wasm_exec.js; streaming fetch for 9.4MB WASM; go.run() without await
- Custom rules injected via parameter to avoid circular deps; dedup keys on line:column not ruleId
- actionlintMeta factory returns no-op check() — metadata rules for documentation/enrichment only
- GHA scorer reads severity directly from GhaUnifiedViolation (no rule lookup map needed)
- Worker onResult callback uses refs (not closures) for correct Worker reuse
- Two-pass dagre layout: dagre LR for triggers+jobs, manual steps inside job group containers
- Null byte check for garbage lz-string decompression (lz-string returns non-null garbage, not null)

**Issues Resolved:**
- WASM binary acquisition: pre-built playground binary downloaded, no Go toolchain required
- Two-pass diagnostic overlap: dedup keys on line:column pairs
- Inlined ucs2length runtime function to eliminate all require() from compiled GHA schema validator
- Stale closure capture fixed with refs pattern for Worker reuse

**Issues Deferred:**
- Bulk publishing 90+ template-similar pages — monitoring for SpamBrain classification post-deploy
- shellcheck/pyflakes WASM integration (documented limitation on GA-L017/GA-L018 rule pages)

**Technical Debt Incurred:**
- None significant
