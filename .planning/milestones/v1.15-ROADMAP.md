## Milestone Summary

**Key Decisions:**
- Used astro/zod import (not bare zod) matching existing schema patterns
- guide.json is a JSON array with id field for file() loader compatibility
- GuideLayout extends Layout.astro directly (two-column grid)
- Sidebar uses CSS position:sticky (zero JS)
- Landing page uses Layout (not GuideLayout) since it is not a chapter page
- Mirrored plot-base.ts pattern for diagram-base.ts
- Default templateRepo and versionTag hardcoded in CodeFromRepo props
- Static dagre layout pre-computed at module level for deployment topology
- Guide OG cache uses separate CACHE_DIR (og-guide) to avoid collisions with EDA cache
- Created /guides/index.astro hub page for future-proofing

**Issues Resolved:**
- Template repo tagging: fastapi-template tagged with v1.0.0 before content authoring
- Header nav strategy: added "Guides" as position 6 link (after EDA, before Tools)

**Issues Deferred:**
None.

**Technical Debt Incurred:**
None.
