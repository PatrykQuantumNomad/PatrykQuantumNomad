# Milestone v1.18: AI Landscape Explorer

**Status:** SHIPPED 2026-03-27
**Phases:** 102-110
**Total Plans:** 25

## Overview

Interactive AI Landscape Explorer with D3 force-directed graph, 51 concept pages, guided tours, compare mode, VS pages, and full site integration. The explorer provides an interactive, explorable AI Landscape map that helps non-technical users understand how AI concepts relate to each other — from broad AI fields down to specific models and frameworks. Built on a 51-node educational dataset with two-tier content (simple + technical descriptions) organized into 9 clusters covering AI, ML, NN, DL, GenAI, Agentic AI, DevTools, Levels, and MCP.

## Phases

### Phase 102: Data Foundation

**Goal**: Establish the canonical data model that every downstream phase builds on — graph, pages, panel, and SEO all consume this data
**Depends on**: Nothing (first phase of v1.18)
**Requirements**: DATA-01, DATA-02, DATA-03, DATA-04, DATA-05
**Success Criteria** (what must be TRUE):
  1. A TypeScript/JSON data model exists with 51 concept nodes (plus ~32 grouped examples) containing slug, name, cluster membership, and edge relationships extracted from the DOT file
  2. Zod schema validates every node with required fields: slug, name, cluster, simpleDescription, technicalDescription; relationships computed via getNodeRelationships() helper from edges array
  3. Every node has two tiers of hand-crafted educational content — a plain-English "simple" description and a "technical" description — each at least 100 words
  4. Edge data preserves labeled relationship types from the DOT file ("subset of", "enables", "e.g.", "powers", "characterized by") across 66 edges
  5. Content collection is registered in Astro config with file() loader and a test build confirms all nodes are accessible via getCollection('aiLandscape')

**Details:**

Two-file data split: nodes.json (file() loader content collection) + graph.json (clusters/edges, direct import). 9 clusters with light/dark hex color pairs. 66 edges preserving 10 relationship types (hierarchy, includes, enables, example, relates, progression, characterizes, aspires, applies, standardizes). 380-test content quality suite validates word counts, schema compliance, cluster coverage, and edge-node referential integrity.

## Milestone Summary

**Key Decisions:**
- D3 micro-module imports for graph (d3-force, d3-zoom) — bundle isolation, never full d3 package (~17KB gzipped)
- Pre-computed layout.json from headless d3-force — deterministic, instant first paint as static SVG
- React island with client:only="react" for graph — D3-zoom + React state cannot be SSR'd
- Modifier key guard for zoom (Ctrl/Cmd + wheel) — prevents accidental page scroll trap
- CSS class-based cluster coloring with html.dark overrides — theme-responsive without SVG regeneration
- Two-file data split: nodes.json (content collection) + graph.json (clusters/edges, direct import)
- Tour data in TypeScript (not JSON) — interface co-location and narrative strings
- 12 curated VS pairs (not all 1,275 permutations) — focus on common confusions, not combinatorial explosion
- GSAP edge pulse with prefers-reduced-motion skip — decorative animation respects accessibility
- MiniMap read-only (no click-to-pan) with aria-hidden=true — visual orientation aid, not interactive
- Homepage card uses inline SVG node-graph motif via frontmatter IIFE + set:html
- LLMs.txt compact cluster listing format (count + first 3 node names)

**Issues Resolved:**
- Build-time vs runtime simulation decision resolved: pre-computed layout.json wins for deterministic SSR
- Content authoring for 51 nodes completed as longest-pole constraint (writing, not engineering)
- Viewport-relative math required for d3-zoom programmatic transitions (viewBox dimensions, not pixel getBoundingClientRect)
- URL sync uses history.replaceState (not pushState) to avoid polluting browser history
- prevTourActive useRef pattern detects tour exit without firing handleClosePanel on initial mount

**Issues Deferred:**
- Hub page (/guides/) uses Tailwind dynamic class interpolation for accentColor — tech debt from v1.16
- Beauty Index OG image "Cannot find module renderers.mjs" error (pre-existing)

**Technical Debt Incurred:**
- None significant — all v1.18 features use established patterns
