# Milestone v1.12: Dockerfile Rules Expansion

**Status:** ✅ SHIPPED 2026-03-02
**Phases:** 72-74
**Total Plans:** 3

## Overview

Expand the Dockerfile Analyzer from 44 to 46 rules by adding PG011 (missing USER directive security rule) and PG012 (Node.js pointer compression efficiency rule). Each rule gets a documentation page. All hardcoded rule counts are updated site-wide and verified with a full production build.

## Phases

### Phase 72: PG011 Missing USER Directive

**Goal**: Users running the Dockerfile Analyzer see a security warning when their final build stage has no USER directive
**Depends on**: Nothing (first phase of v1.12)
**Plans**: 1 plan

Plans:
- [x] 72-01: Implement PG011 rule and register in engine

**Details:**
- PG011 security rule detects missing USER directive in final Dockerfile stage
- Warning severity matching DL3002 convention
- Non-overlap boundary: PG011 fires only when zero USER instructions exist, DL3002 handles all cases where USER is present
- Absence-detection pattern: checks for zero instructions of a type in final stage, flags on FROM line
- FROM scratch with no USER does NOT trigger PG011
- 5 edge-case Vitest scenarios all pass
- Documentation page auto-generates at /tools/dockerfile-analyzer/rules/pg011/

### Phase 73: PG012 Node.js Pointer Compression

**Goal**: Users running the Dockerfile Analyzer on Node.js images see an efficiency suggestion for platformatic/node-caged with memory savings context
**Depends on**: Phase 72
**Plans**: 1 plan

Plans:
- [x] 73-01: Implement PG012 rule, tests, and register in engine

**Details:**
- PG012 info-level efficiency rule suggests platformatic/node-caged for V8 pointer compression (~50% memory reduction)
- Exact string equality for image name matching prevents false positives on nodeconfig, nodejs-slim, etc.
- Registry check mandatory: getRegistry() must be null or docker.io to exclude custom registry node images
- Explanation covers Node 25+ version requirement, ~50% memory benefit, 4GB isolate limit, N-API addon compatibility
- 7 Vitest scenarios verify all edge cases: official node, python, custom registry, multi-stage, scratch, alias references
- Rule registered in allRules (45 -> 46 rules, efficiency 8 -> 9)
- Documentation page auto-generated at /tools/dockerfile-analyzer/rules/pg012/
